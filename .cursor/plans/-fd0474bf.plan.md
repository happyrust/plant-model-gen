# 创建房间关系构建和元件计算测试

## 目标

创建一个完整的测试案例，验证 `build_room_relations_v2` 的完整流程，包括：

1. 查询所有房间信息
2. 生成房间模型（使用 `gen_all_geos_data`）
3. 生成BRAN模型（使用 `gen_all_geos_data`）
4. 构建房间关系
5. 计算每个房间包含了哪些其他的元件（构件）

## 测试文件位置

- 文件路径: `gen-model-fork/tests/test_room_build_and_calculation_v2.rs`
- 独立测试文件，不需要添加到模块（tests/ 目录下的文件会自动被 cargo test 识别）

## 测试步骤

### 1. 初始化数据库连接

- 使用 `init_test_surreal()` 初始化 SurrealDB 连接
- 加载 `DbOption` 配置（从 `DbOption.toml`）
- 将 `DbOption` 转换为 `DbOptionExt`：`DbOptionExt::from(db_option.clone())`

### 2. 查询所有房间信息

- 使用 `aios_core::room::algorithm::query_all_room_infos()` 查询所有房间
- 传入 `db_option.get_room_key_word()` 作为关键词过滤
- 打印查询到的房间数量和基本信息
- 提取所有房间面板 refno 列表

### 3. 生成所有房间模型

- 使用 `crate::fast_model::gen_model::gen_all_geos_data()` 生成所有房间面板的几何模型
- 传入参数：`(房间面板 refno 列表, &db_option_ext, None, None)`
- 等待模型生成完成
- 打印生成进度和结果

### 4. 查询所有BRAN

- 查询数据库中所有的BRAN refno（参考 `bran_room_real_data_test.rs` 中的 `query_all_brans_complete()` 方法）
- 打印BRAN数量和基本信息

### 5. 生成所有BRAN模型

- 使用 `crate::fast_model::gen_model::gen_all_geos_data()` 生成所有BRAN的几何模型
- 传入参数：`(BRAN refno 列表, &db_option_ext, None, None)`
- 等待模型生成完成
- 打印生成进度和结果

### 6. 构建房间关系

- 调用 `crate::fast_model::room_model::build_room_relations(db_option)` 
- 等待构建完成并获取统计信息 `RoomBuildStats`
- 打印构建统计：房间数、面板数、构件数、耗时、缓存命中率等

### 7. 计算每个房间包含的元件

- 遍历所有房间
- 对每个房间，查询 `room_relate` 关系，获取所有关联的构件
- 使用 SQL 查询：
  ```sql
  SELECT 
      noun,
      COUNT() as count
  FROM pe 
  WHERE room_relate CONTAINS <room_refno>
  AND solid = true
  GROUP BY noun
  ORDER BY noun
  ```

- 统计每个房间的构件数量和类型分布（按 noun 分组）
- 打印每个房间的元件统计信息
- 可选：生成汇总报告（房间号、构件总数、各类型构件数量）

## 依赖项

- `aios_core::room::algorithm::query_all_room_infos` - 查询房间信息
- `crate::fast_model::room_model::build_room_relations` - 构建房间关系
- `crate::fast_model::gen_model::gen_all_geos_data` - 生成几何模型
- `crate::options::DbOptionExt` - 数据库配置扩展（需要从 `DbOption` 转换）

## 查询房间元件的SQL示例

```sql
SELECT 
    noun,
    COUNT() as count
FROM pe 
WHERE room_relate CONTAINS <room_refno>
AND solid = true
GROUP BY noun
ORDER BY noun
```

## 注意事项

- 测试需要启用 `sqlite` 特性
- 需要有效的 `DbOption.toml` 配置文件
- 测试可能需要较长时间（几分钟到十几分钟），因为涉及大量模型生成
- 确保测试数据中有足够的房间、面板和BRAN数据
- `gen_all_geos_data` 需要 `DbOptionExt` 类型，需要从 `DbOption` 转换：`DbOptionExt::from(db_option.clone())`
- 函数签名：`gen_all_geos_data(manual_refnos: Vec<RefnoEnum>, db_option: &DbOptionExt, incr_updates: Option<IncrGeoUpdateLog>, target_sesno: Option<u32>) -> Result<bool>`
- 使用真实数据库数据，不是模拟数据
