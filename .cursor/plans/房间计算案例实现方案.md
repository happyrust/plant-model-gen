# 房间计算案例实现方案

## 目标

在 `/Volumes/DPC/work/plant-code/gen-model-fork/examples/` 目录下创建一个完整的房间计算案例示例程序，展示：

1. 模型生成 (`gen_all_geos_data`)
2. 构建房间关系 (`build_room_relations_v2`)
3. 房间查询方法 (`query_room_by_point_v2`, `batch_query_room_numbers`)

## 实现内容

### 1. 创建示例程序文件

**文件路径**: `examples/room_calculation_demo.rs`

**主要功能模块**:

- **初始化模块**: 初始化 SurrealDB 连接和配置
- **模型生成模块**: 演示如何生成指定 refno 的模型数据
- **关系构建模块**: 演示如何构建房间面板关系和房间构件关系
- **查询模块**: 演示点查询和批量查询房间的方法
- **验证模块**: 验证生成的数据和关系的正确性

### 2. 关键实现点

#### 2.1 初始化配置

- 使用 `get_db_option()` 获取数据库配置
- 使用 `init_test_surreal()` 或 `init_demo_test_surreal()` 初始化 SurrealDB
- 配置房间关键词（如 `["-RM", "-ROOM"]`）
- 设置模型生成选项（`gen_mesh`, `gen_model`, `apply_boolean_operation`）

#### 2.2 模型生成

- 调用 `gen_all_geos_data(refnos, &db_option_ext, None, None)` 
- 传入需要生成模型的 refno 列表
- 处理生成结果和错误

#### 2.3 关系构建

- 调用 `build_room_relations_v2(&db_option)` 构建房间关系
- 获取构建统计信息（房间数、面板数、构件数、耗时等）
- 验证关系是否正确保存到数据库

#### 2.4 房间查询

- 使用 `query_room_number_by_point_v2(point)` 进行点查询
- 使用 `batch_query_room_numbers(points, batch_size)` 进行批量查询
- 使用 `query_room_panel_by_point_v2(point)` 查询面板信息
- 展示查询结果和性能统计

### 3. 代码结构

```rust
// 示例程序结构
async fn main() -> Result<()> {
    // 1. 初始化
    let db_option = init_database()?;
    
    // 2. 模型生成示例
    demo_model_generation(&db_option).await?;
    
    // 3. 关系构建示例
    demo_build_relations(&db_option).await?;
    
    // 4. 房间查询示例
    demo_room_queries().await?;
    
    // 5. 验证结果
    verify_results().await?;
    
    Ok(())
}
```

### 4. 依赖和特性

- 需要启用 `sqlite` 特性（用于房间查询）
- 需要启用 `gen_model` 特性（用于模型生成）
- 使用 `tokio` 运行时
- 使用 `anyhow` 进行错误处理

### 5. 测试数据

- 使用配置中的房间关键词查询房间
- 使用示例 refno 进行模型生成
- 使用示例坐标点进行房间查询

### 6. 输出信息

- 每个步骤的执行状态和耗时
- 生成/处理的记录数量
- 查询结果和性能指标
- 错误信息和警告
