# 异地协同与异地运维 - 用户操作指南

## 概述

本系统提供两个相关但独立的功能模块：

1. **异地协同** (`/collaboration`) - 多站点配置共享和协调
2. **异地运维** (`/remote-sync`) - 数据同步和运维管理

---

## 一、异地协同 vs 异地运维

### 异地协同 (Collaboration)
**目的**: 管理多个站点之间的配置共享和任务协调

**核心功能**:
- 创建协同组（多站点组合）
- 配置共享策略
- 任务协调
- 双向/单向同步配置

**使用场景**:
- 多个站点需要共享相同配置
- 跨站点任务协调
- 配置一致性管理

### 异地运维 (Remote Sync)
**目的**: 管理中心站点与远程站点之间的数据同步

**核心功能**:
- 环境配置（MQTT、文件服务器）
- 站点管理
- 数据同步监控
- 性能分析
- 日志查询
- 告警通知

**使用场景**:
- 中心站点向远程站点推送数据
- 远程站点向中心站点拉取数据
- 实时监控同步状态
- 故障排查和性能优化

## 二、异地协同操作流程

### 2.1 创建协同组

#### 步骤 1: 进入异地协同页面
```
导航栏 → 工具 → 异地协同
```

#### 步骤 2: 点击"创建协同组"
填写以下信息：
- **协同组名称**: 例如 "生产环境配置组"
- **描述**: 协同组的用途说明
- **协同类型**:
  - 配置共享 (ConfigSharing)
  - 数据同步 (DataSync)
  - 任务协调 (TaskCoordination)
  - 混合模式 (Hybrid)
- **同步策略**:
  - 单向 (OneWay): A → B
  - 双向 (TwoWay): A ↔ B
  - 手动 (Manual): 手动触发
- **位置**: 地理位置标识

#### 步骤 3: 添加站点
- 选择要加入协同组的站点
- 配置每个站点的角色（主站点/从站点）

#### 步骤 4: 保存并激活
- 保存配置
- 激活协同组
- 开始配置同步

### 2.2 管理协同组

#### 查看协同组详情
- 点击协同组卡片的"查看详情"
- 查看站点列表
- 查看同步状态
- 查看同步历史

#### 修改协同组
- 添加/删除站点
- 修改同步策略
- 更新配置

#### 暂停/恢复协同
- 暂停: 停止配置同步
- 恢复: 继续配置同步

## 三、异地运维操作流程

### 3.1 部署新环境（完整流程）

#### 步骤 1: 进入部署向导
```
导航栏 → 工具 → 异地运维 → 部署向导
```

#### 步骤 2: 配置环境信息
**环境基本信息**:
- **环境名称**: 例如 "北京生产环境"
- **位置**: 例如 "北京"
- **数据库编号**: 例如 "7999,8000"

**MQTT 配置**:
- **MQTT 主机**: 例如 "mqtt://192.168.1.100:1883"
- **MQTT 端口**: 默认 1883
- **用户名/密码**: 如果需要认证

**文件服务器配置**:
- **文件服务器地址**: 例如 "http://192.168.1.100:8080"
- **下载路径**: 例如 "/assets/archives"

**重连策略**:
- **初始重连间隔**: 默认 1000ms
- **最大重连间隔**: 默认 30000ms

#### 步骤 3: 添加站点
**站点基本信息**:
- **站点名称**: 例如 "远程站点 A"
- **位置**: 例如 "上海"
- **数据库编号**: 例如 "7999"

**站点连接配置**:
- **HTTP 地址**: 站点的 HTTP 访问地址
- **备注**: 站点说明

#### 步骤 4: 配置拓扑关系
- 使用拓扑配置页面可视化配置
- 拖拽环境和站点节点
- 连接环境到站点（表示同步关系）

#### 步骤 5: 测试连接
- 测试 MQTT 连接
- 测试文件服务器连接
- 测试站点 HTTP 连接

#### 步骤 6: 激活环境
- 点击"激活环境"
- 系统开始监听 MQTT 消息
- 开始数据同步

### 3.2 使用拓扑配置

#### 进入拓扑配置页面
```
导航栏 → 工具 → 异地运维 → 拓扑配置
```

#### 可视化编辑
1. **添加环境节点**
   - 点击"添加环境"
   - 拖动节点到合适位置

2. **添加站点节点**
   - 点击"添加站点"
   - 拖动节点到合适位置

3. **连接节点**
   - 从环境节点拖动到站点节点
   - 建立同步关系

4. **自动布局**
   - 点击"自动布局"
   - 系统自动排列节点

5. **保存配置**
   - 点击"保存"
   - 配置写入数据库

### 3.3 监控同步状态

#### 监控仪表板
```
导航栏 → 工具 → 异地运维 → 监控仪表板
```

**实时信息**:
- 环境状态（运行/停止/暂停）
- MQTT 连接状态
- 活跃任务数量
- 队列长度

**操作**:
- 启动/停止同步服务
- 暂停/恢复同步
- 清空队列
- 添加手动任务

### 3.4 查询日志

#### 进入日志查询
```
导航栏 → 工具 → 异地运维 → 日志查询
```

**筛选条件**:
- 关键词搜索
- 状态筛选（完成/失败/运行中）
- 环境筛选
- 方向筛选（推送/拉取）

**查看详情**:
- 点击日志条目
- 查看完整信息
- 查看错误详情

**导出日志**:
- 点击"导出 CSV"或"导出 JSON"
- 最多导出 10000 条

### 3.5 性能分析

#### 进入性能监控
```
导航栏 → 工具 → 异地运维 → 性能监控
```

**实时指标**:
- 同步速率（MB/s）
- 成功率（%）
- 完成统计
- 系统资源（CPU/内存）

**历史趋势**:
- 任务统计图表
- 数据量趋势
- 耗时分析
- P50/P95/P99 统计

**导出报告**:
- 点击"导出报告"
- 生成 CSV 文件

### 3.6 告警管理

#### 进入告警中心
```
导航栏 → 工具 → 异地运维 → 告警中心
```

**查看告警**:
- 实时告警通知
- 告警级别（严重/错误/警告/信息）
- 告警时间
- 告警详情

**配置告警规则**:
- 失败率阈值（默认 30%）
- 队列积压阈值（默认 100）
- MQTT 重连阈值（默认 5 次）

**通知渠道**:
- 界面通知（默认开启）
- 邮件通知
- Webhook

## 四、典型使用场景

### 场景 1: 中心站点向多个远程站点推送数据

#### 配置步骤
1. 创建环境（中心站点）
2. 添加多个站点（远程站点）
3. 配置拓扑关系（中心 → 远程 A, 远程 B, 远程 C）
4. 激活环境
5. 监控同步状态

#### 数据流向
```
中心站点 (生成 CBA 文件)
    ↓ MQTT 通知
远程站点 A, B, C (接收通知)
    ↓ HTTP 下载
中心站点文件服务器 (/assets/archives)
```

### 场景 2: 远程站点向中心站点拉取数据

#### 配置步骤
1. 创建环境（远程站点）
2. 配置中心站点的文件服务器地址
3. 配置拉取策略
4. 激活环境
5. 监控同步状态

### 场景 3: 多站点配置共享

#### 使用异地协同
1. 创建协同组
2. 添加所有需要共享配置的站点
3. 选择"配置共享"类型
4. 选择"双向"同步策略
5. 激活协同组

## 五、故障排查

### 问题 1: MQTT 连接失败

**症状**: 告警中心显示"MQTT 连接持续失败"

**排查步骤**:
1. 检查 MQTT 服务器是否运行
2. 检查网络连接
3. 检查 MQTT 配置（主机、端口、认证）
4. 查看日志详情

**解决方案**:
- 修正 MQTT 配置
- 重启 MQTT 服务器
- 检查防火墙规则

### 问题 2: 队列积压

**症状**: 告警中心显示"队列积压严重"

**排查步骤**:
1. 查看性能监控页面
2. 检查同步速率
3. 查看失败任务

**解决方案**:
- 暂停同步，清空队列
- 增加并发数
- 优化网络带宽

### 问题 3: 同步失败率高

**症状**: 告警中心显示"同步失败率过高"

**排查步骤**:
1. 查看日志查询页面
2. 筛选失败日志
3. 查看错误详情

**解决方案**:
- 修复网络问题
- 检查文件服务器
- 重试失败任务

## 八、常见问题 (FAQ)

### Q1: 异地协同和异地运维有什么区别？
**A**: 异地协同用于多站点配置共享，异地运维用于数据同步和运维管理。

### Q2: 如何添加新的远程站点？
**A**: 进入"异地运维 → 环境列表"，选择环境，点击"添加站点"。

### Q3: 如何查看同步失败的原因？
**A**: 进入"日志查询"，筛选失败状态，点击日志查看详情。

### Q4: 如何导出性能报告？
**A**: 进入"性能监控"，点击"导出报告"按钮。

### Q5: 告警通知如何配置？
**A**: 进入"告警中心"，点击"配置"按钮，设置阈值和通知渠道。
