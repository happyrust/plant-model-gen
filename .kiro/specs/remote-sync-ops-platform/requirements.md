# Requirements Document

## Introduction

本需求文档定义了将前端 Web UI 升级为完整的异地协同运维平台的功能需求。该平台将整合现有的异地同步功能（MQTT + 文件监控）与前端管理界面，提供统一的部署、配置、监控和运维能力，使运维人员能够通过 Web 界面完成异地协同系统的全生命周期管理。

当前系统已实现：
- 后端异地同步核心功能（MQTT 实时通知 + 文件监控 + HTTP 上传）
- 基础的环境和站点管理 API
- 前端协同组列表和创建界面（基于适配器模式）

需要增强的能力：
- 完整的部署流程管理
- 实时同步状态监控
- 数据流向可视化
- 性能指标监控
- 告警和日志管理
- 运维操作工具集

### 关键工作流程

#### 1. 部署新环境流程
```
运维人员 → 访问部署向导
         ↓
步骤1: 填写环境信息 (名称, MQTT, 文件服务器, 地区)
         ↓
步骤2: 添加远程站点 (名称, HTTP 地址, 数据库编号)
         ↓
步骤3: 连接测试 (测试 MQTT 和 HTTP 连接)
         ↓
步骤4: 激活环境 (写入 DbOption.toml, 启动运行时)
         ↓
运维平台 → 显示激活成功, 跳转到监控仪表板
```

#### 2. 监控同步状态流程
```
运维人员 → 访问监控仪表板
         ↓
建立 SSE 连接 (/api/sync/events)
         ↓
接收实时事件:
  - SyncStarted: 任务开始
  - SyncProgress: 进度更新
  - SyncCompleted: 任务完成
  - SyncFailed: 任务失败
  - MqttConnected: MQTT 连接
  - MqttDisconnected: MQTT 断开
         ↓
更新界面显示 (任务列表, 性能指标, 告警通知)
```

#### 3. 排查同步问题流程
```
运维人员 → 发现任务失败告警
         ↓
点击告警 → 跳转到日志查询页面
         ↓
自动填充筛选条件 (状态=失败, 时间=最近1小时)
         ↓
查看失败任务详情 (错误信息, 重试历史)
         ↓
分析错误原因:
  - 网络连接问题 → 测试 HTTP 连接
  - MQTT 断开 → 检查 MQTT 状态
  - 文件权限问题 → 检查目标路径
         ↓
执行修复操作:
  - 重启同步服务
  - 手动重试任务
  - 调整配置参数
         ↓
验证修复结果 → 监控后续任务状态
```

## Glossary

- **RemoteSyncEnv（远程同步环境）**: 代表一个地理位置或逻辑分组的同步环境，包含 MQTT 配置、文件服务器地址等
- **RemoteSyncSite（远程站点）**: 环境下的具体外部站点，用于从其他地区获取数据
- **SyncControlCenter（同步控制中心）**: 全局单例，管理整个同步服务的生命周期和状态
- **SyncTask（同步任务）**: 单个文件的同步任务，包含状态、优先级、重试次数等信息
- **SiteMetadata（站点元数据）**: 描述站点同步文件的元数据信息，存储在 metadata.json 文件中
- **MQTT（消息队列遥测传输）**: 轻量级的发布/订阅消息传输协议，用于实时通知文件变更
- **SSE（Server-Sent Events）**: 服务器推送事件，用于实时推送同步状态更新到前端
- **CBA 文件**: 压缩的增量数据包文件，包含数据库变更记录
- **运维平台（Ops Platform）**: 前端 Web UI 系统，提供异地协同的部署、监控和管理功能

## Requirements

### Requirement 1: 部署流程管理

**User Story:** 作为运维人员，我希望通过 Web 界面完成异地协同环境的部署配置，以便快速建立多地区的数据同步网络

#### Acceptance Criteria

1. WHEN 运维人员访问部署向导页面，THE 运维平台 SHALL 显示分步骤的部署配置界面，包含环境创建、站点添加、连接测试和激活确认四个步骤
2. WHEN 运维人员填写环境基本信息（名称、MQTT 地址、文件服务器地址、地区标识），THE 运维平台 SHALL 验证输入格式的正确性，并在提交前显示配置预览
3. WHEN 运维人员添加远程站点配置，THE 运维平台 SHALL 支持批量导入站点信息，并自动验证 HTTP 地址的可达性
4. WHEN 运维人员点击连接测试按钮，THE 运维平台 SHALL 调用后端 API 测试 MQTT 连接和文件服务器连接，并显示测试结果和延迟信息
5. WHEN 运维人员确认激活环境，THE 运维平台 SHALL 调用激活 API，写入 DbOption.toml 配置文件，启动 MQTT 订阅和文件监控服务，并显示激活进度

### Requirement 2: 实时同步状态监控

**User Story:** 作为运维人员，我希望实时查看各个环境和站点的同步状态，以便及时发现和处理同步异常

#### Acceptance Criteria

1. WHEN 运维人员访问监控仪表板页面，THE 运维平台 SHALL 通过 SSE 连接实时接收同步事件，并在 1 秒内更新界面显示
2. WHEN 同步任务状态发生变化（Pending/Running/Completed/Failed），THE 运维平台 SHALL 在任务列表中更新对应任务的状态标识和进度条
3. WHEN 环境的 MQTT 连接状态变化，THE 运维平台 SHALL 在环境卡片上显示连接状态指示器（绿色已连接/红色断开/黄色重连中）
4. WHEN 运维人员选择特定环境，THE 运维平台 SHALL 显示该环境下所有站点的同步统计信息，包括成功数、失败数、待处理数和同步速率
5. WHEN 同步任务失败超过 3 次，THE 运维平台 SHALL 在界面顶部显示告警通知，并提供快速跳转到错误详情的链接

### Requirement 3: 数据流向可视化

**User Story:** 作为运维人员，我希望通过可视化图表了解数据在各站点间的流向和流量，以便优化同步策略和排查瓶颈

#### Acceptance Criteria

1. WHEN 运维人员访问流向图页面，THE 运维平台 SHALL 使用力导向图或桑基图展示环境和站点之间的数据流向关系
2. WHEN 运维人员将鼠标悬停在流向连线上，THE 运维平台 SHALL 显示该流向的详细统计信息，包括文件数量、总大小、平均速率和最近同步时间
3. WHEN 运维人员点击站点节点，THE 运维平台 SHALL 高亮显示该站点相关的所有流向，并在侧边栏显示站点详细信息
4. WHEN 运维人员选择时间范围筛选器，THE 运维平台 SHALL 根据选定时间范围（最近 1 小时/24 小时/7 天/30 天）重新计算和渲染流向图
5. WHEN 数据流向中存在异常（如单向流量过大或长时间无数据），THE 运维平台 SHALL 在流向连线上显示警告标识

### Requirement 4: 性能指标监控

**User Story:** 作为运维人员，我希望监控同步系统的性能指标，以便评估系统健康状况和容量规划

#### Acceptance Criteria

1. WHEN 运维人员访问性能监控页面，THE 运维平台 SHALL 显示实时性能指标，包括同步速率（MB/s）、任务队列长度、活跃任务数和系统资源使用率
2. WHEN 运维人员选择时间范围，THE 运维平台 SHALL 使用折线图展示历史性能趋势，支持最近 1 小时、24 小时、7 天和 30 天的数据查询
3. WHEN 性能指标超过预设阈值（如队列长度 > 100 或同步速率 < 1 MB/s），THE 运维平台 SHALL 在指标卡片上显示警告状态
4. WHEN 运维人员点击性能指标卡片，THE 运维平台 SHALL 展开详细的性能分析面板，显示分位数统计（P50/P95/P99）和异常时间点标注
5. WHEN 运维人员导出性能报告，THE 运维平台 SHALL 生成包含图表和统计数据的 PDF 或 CSV 文件

### Requirement 5: 同步日志管理

**User Story:** 作为运维人员，我希望查询和分析同步日志，以便排查历史问题和审计同步操作

#### Acceptance Criteria

1. WHEN 运维人员访问日志查询页面，THE 运维平台 SHALL 提供多维度筛选条件，包括环境、站点、状态、方向、时间范围和关键词搜索
2. WHEN 运维人员提交日志查询请求，THE 运维平台 SHALL 在 2 秒内返回查询结果，并支持分页显示（每页 50 条，最多 500 条）
3. WHEN 运维人员点击日志条目，THE 运维平台 SHALL 展开显示完整的日志详情，包括文件路径、大小、记录数、耗时、错误信息和重试历史
4. WHEN 运维人员导出日志，THE 运维平台 SHALL 支持导出当前筛选结果为 CSV 或 JSON 格式，并限制单次导出不超过 10000 条
5. WHEN 日志中包含错误信息，THE 运维平台 SHALL 高亮显示错误关键词，并提供错误码的解释和建议解决方案

### Requirement 6: 站点元数据浏览

**User Story:** 作为运维人员，我希望浏览和管理站点的同步文件元数据，以便了解各站点的数据状态和手动触发同步

#### Acceptance Criteria

1. WHEN 运维人员访问站点详情页面，THE 运维平台 SHALL 显示该站点的元数据信息，包括环境名称、站点名称、生成时间和文件条目列表
2. WHEN 运维人员点击刷新元数据按钮，THE 运维平台 SHALL 调用后端 API 强制刷新元数据（refresh=true），并在 5 秒内显示最新数据
3. WHEN 运维人员浏览文件条目列表，THE 运维平台 SHALL 显示每个文件的名称、大小、哈希值、记录数、方向、更新时间和下载链接
4. WHEN 运维人员点击文件下载链接，THE 运维平台 SHALL 通过后端代理下载文件，并显示下载进度条
5. WHEN 元数据获取失败，THE 运维平台 SHALL 显示错误原因（本地路径不存在/HTTP 不可达/缓存过期），并提供重试和使用缓存的选项

### Requirement 7: 运维操作工具

**User Story:** 作为运维人员，我希望通过 Web 界面执行常见的运维操作，以便快速响应系统问题和维护需求

#### Acceptance Criteria

1. WHEN 运维人员点击启动同步服务按钮，THE 运维平台 SHALL 调用启动 API，并在 3 秒内显示服务启动结果和运行状态
2. WHEN 运维人员点击停止同步服务按钮，THE 运维平台 SHALL 弹出确认对话框，确认后调用停止 API，并等待所有运行中的任务完成或超时（最长 30 秒）
3. WHEN 运维人员点击暂停同步按钮，THE 运维平台 SHALL 暂停新任务的处理，但不中断正在运行的任务，并在界面上显示暂停状态标识
4. WHEN 运维人员点击清空任务队列按钮，THE 运维平台 SHALL 弹出二次确认对话框，确认后清空所有 Pending 状态的任务，并显示清空数量
5. WHEN 运维人员手动添加同步任务，THE 运维平台 SHALL 提供表单输入文件路径、大小、优先级等参数，验证后调用添加任务 API

### Requirement 8: 告警和通知

**User Story:** 作为运维人员，我希望在系统出现异常时收到及时的告警通知，以便快速响应和处理问题

#### Acceptance Criteria

1. WHEN 同步任务失败率超过 10%（最近 100 个任务），THE 运维平台 SHALL 在界面顶部显示警告横幅，并记录告警事件
2. WHEN MQTT 连接断开超过 5 分钟，THE 运维平台 SHALL 显示严重告警，并提供重连和查看日志的快捷操作
3. WHEN 任务队列积压超过 50 个任务，THE 运维平台 SHALL 显示容量告警，并建议增加并发数或检查网络状况
4. WHEN 运维人员点击告警通知，THE 运维平台 SHALL 跳转到相关的详情页面（如日志查询页面并自动填充筛选条件）
5. WHEN 运维人员配置告警规则，THE 运维平台 SHALL 支持自定义阈值和通知方式（界面通知/邮件/Webhook），并保存到配置文件

### Requirement 9: 配置管理

**User Story:** 作为运维人员，我希望通过 Web 界面管理同步系统的配置参数，以便动态调整系统行为而无需重启服务

#### Acceptance Criteria

1. WHEN 运维人员访问配置管理页面，THE 运维平台 SHALL 显示当前的同步配置，包括自动重试、最大重试次数、重试延迟、最大并发数、批量大小和同步间隔
2. WHEN 运维人员修改配置参数，THE 运维平台 SHALL 实时验证参数的有效性（如最大并发数 1-20，重试次数 0-10），并显示验证错误
3. WHEN 运维人员保存配置，THE 运维平台 SHALL 调用更新配置 API，并在配置生效后显示成功提示
4. WHEN 运维人员点击重置为默认值按钮，THE 运维平台 SHALL 弹出确认对话框，确认后恢复所有配置为系统默认值
5. WHEN 配置更新失败，THE 运维平台 SHALL 显示详细的错误信息，并保持界面上的配置为修改前的状态

### Requirement 10: 多环境管理

**User Story:** 作为运维人员，我希望在一个界面中管理多个异地协同环境，以便统一监控和操作不同地区的同步系统

#### Acceptance Criteria

1. WHEN 运维人员访问环境列表页面，THE 运维平台 SHALL 显示所有已配置的环境，并标识当前激活的环境
2. WHEN 运维人员切换激活环境，THE 运维平台 SHALL 调用激活 API，停止当前环境的运行时，启动新环境的运行时，并在 10 秒内完成切换
3. WHEN 运维人员比较多个环境的配置，THE 运维平台 SHALL 并排显示环境配置的差异项，并高亮不同的配置值
4. WHEN 运维人员复制环境配置，THE 运维平台 SHALL 创建新环境并复制源环境的所有配置（除 ID 和名称外），并提示修改必要的参数
5. WHEN 运维人员删除环境，THE 运维平台 SHALL 弹出确认对话框，显示该环境关联的站点数量，确认后级联删除所有关联站点和日志记录

### Requirement 11: 可视化拓扑配置

**User Story:** 作为运维人员，我希望通过可视化的 Canvas 画布配置主节点和同步节点之间的连接关系，以便直观地设计和管理复杂的同步拓扑结构

#### Acceptance Criteria

1. WHEN 运维人员访问拓扑配置页面，THE 运维平台 SHALL 显示一个可交互的 Canvas 画布，支持拖拽、缩放和平移操作
2. WHEN 运维人员在画布上创建环境节点（主节点），THE 运维平台 SHALL 允许配置环境属性：名称、MQTT 地址和端口、文件服务器地址、地区标识、本地数据库编号列表、MQTT 重连参数
3. WHEN 运维人员在画布上创建站点节点（同步节点），THE 运维平台 SHALL 允许配置站点属性：名称、所属环境、地区标识、HTTP 地址、数据库编号列表、备注信息
4. WHEN 运维人员连接环境节点到站点节点，THE 运维平台 SHALL 创建有向连线表示同步关系，并自动设置站点的 env_id 关联
5. WHEN 运维人员保存拓扑配置，THE 运维平台 SHALL 验证拓扑的有效性（环境节点必须有 MQTT 和文件服务器配置、站点节点必须关联到环境、数据库编号不为空），并将配置转换为 remote_sync_envs 和 remote_sync_sites 表记录
6. WHEN 运维人员加载已有拓扑，THE 运维平台 SHALL 从数据库读取环境和站点配置，自动生成拓扑图并应用层次布局算法（环境节点在上层，站点节点在下层）
7. WHEN 运维人员在画布上选中节点，THE 运维平台 SHALL 在侧边栏显示详细配置面板，支持实时编辑属性并立即反映到画布和数据库
8. WHEN 运维人员删除环境节点，THE 运维平台 SHALL 弹出确认对话框，显示关联的站点数量，确认后级联删除所有关联站点和连线
9. WHEN 运维人员导出拓扑配置，THE 运维平台 SHALL 支持导出为 JSON 格式，包含所有环境、站点和连接关系，并支持从 JSON 文件导入拓扑

### Requirement 12: CBA 文件分发服务

**User Story:** 作为运维人员，我希望系统自动提供 CBA 文件的 HTTP 访问服务，以便远程站点可以通过 HTTP 下载增量数据包

#### Acceptance Criteria

1. WHEN dpcsync 生成 CBA 文件到 assets/archives 目录，THE 运维平台 SHALL 自动通过 HTTP 端点 /assets/archives 提供文件访问服务
2. WHEN 远程站点请求下载 CBA 文件，THE 运维平台 SHALL 支持通过 GET 请求访问 /assets/archives/{filename}.cba，并返回文件内容
3. WHEN 环境配置中设置 file_server_host 参数，THE 运维平台 SHALL 使用该地址作为 CBA 文件的下载 URL 前缀（如 http://host:port/assets/archives）
4. WHEN 生成站点元数据时，THE 运维平台 SHALL 在 metadata.json 中包含完整的 download_url（file_server_host + 相对路径）
5. WHEN 运维人员查看站点元数据，THE 运维平台 SHALL 显示每个 CBA 文件的下载链接，点击后可直接下载文件
