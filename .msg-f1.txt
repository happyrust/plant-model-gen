refactor(model-writer): hoist missing_neg_carriers to trait method (v3 F.1)

Closes the last "type leak" on the trait surface: `WriteBaseReport` no
longer carries `Vec<RefnoEnum>`, and the orchestrator no longer needs a
shared `Arc<Mutex<HashSet>>` to fan-in batch-level carrier sets.

Trait change (additive, default-empty):
- New method: `take_missing_neg_carriers(&self) -> Result<Vec<RefnoEnum>>`
  with a default impl returning `Vec::new()`, so backends that do not
  participate in reconcile (Parquet / DrainOnly / DuckLake-skeleton) get
  it for free.

Backend changes:
- Surreal: holds `Mutex<Vec<RefnoEnum>>` internally; each
  `write_base_batch` appends carriers from the underlying
  `SaveInstanceDataReport`; `take_missing_neg_carriers` drains and returns.
- Mock (RecordingBackend): records the `take_missing_neg_carriers` call
  into its snapshot and drains the `injected_missing_neg` Mutex (so the
  verify binary's existing injection mechanism still works, just routed
  through the new method).
- Compare wrapper: fans out to primary then candidate, returns primary's
  drained Vec; emits `[model-writer:compare] take_missing_neg_diff` when
  lengths disagree.

Report change:
- `WriteBaseReport.missing_neg_carriers` field removed.
- `WriteBaseReport.missing_neg_count` retained for per-batch logging.

Orchestrator simplification:
- `run_base_writer` no longer takes/uses
  `missing_neg_carriers: Arc<Mutex<HashSet<RefnoEnum>>>`.
- Main routine drains via the trait method after every batch has been
  written, then sorts/dedups for reconcile.

Verify binary (exit code 4 still reserved for injection mismatch):
- Asserts `write_base_batch` report no longer carries carriers.
- Asserts `take_missing_neg_carriers` returns the injected vector on the
  first call.
- Asserts the second `take_missing_neg_carriers` returns empty (drain is
  idempotent).
- Compare wrapper test path also exercises `take_missing_neg_carriers`
  and asserts primary (RecordingBackend) snapshot covers it.

Verification (local):
  cargo run --bin verify_model_writer_trait --features model-writer-mock
  -> PASS — 11 mock snapshot entries (incl. two takes), all assertions hold.
