# IDA Pro 中 Snout 生成过程分析 - core.dll

## 概述

本文档详细分析 e3d (AVEVA PDMS/E3D) 的 core.dll 中 Snout（锥台）几何体的生成过程，基于 IDA Pro 反编译代码。

## 1. 函数入口：MBXSNO

### 函数签名
```c
int MBXSNO(int a1, int a2, double *a3, double *a4, double *a5, double *a6, double *a7, int a8)
```

**地址**: `0x108942cc` (函数开始) / `0x108942f8` (实际入口点)

### 参数分析

根据汇编代码和反编译分析，参数含义如下：

| 参数 | 类型 | 含义 | 使用位置 |
|------|------|------|----------|
| `a1` | int | EBP 寄存器值（调用约定） | 栈帧保存 |
| `a2` | int | 变换矩阵或几何上下文 | `CONCAT` 调用 |
| `a3` | double* | **高度参数** (height) | `*a3 * 0.5` → `var_C8+0x58` |
| `a4` | double* | **底部X半径** (bottom_x) | `*a4 * 0.5` → `var_C8+0x48` |
| `a5` | double* | **底部Y半径** (bottom_y) | `*a5 * 0.5` → `var_C8+0x50` |
| `a6` | double* | **顶部X半径** (top_x) | `*a6 * 0.5` → `var_E0` |
| `a7` | double* | **顶部Y半径** (top_y) | `*a7 * 0.5` → `var_D8` |
| `a8` | int | **输出缓冲区** (bounding box) | 传递给 `MBXCIR` |

### 数据结构

函数内部使用的主要局部变量：

```c
_DWORD _90[29];      // [esp+90h] - 变换矩阵（约232字节）
int v9;              // [esp+10h] - 标志位，初始化为 -1
double v10;          // [esp+18h] - 顶部X半径的一半
double v11;          // [esp+20h] - 顶部Y半径的一半
int v12;             // [esp+30h] - 单位矩阵
double v13;          // [esp+78h] - 底部X半径的一半
double v14;          // [esp+80h] - 底部Y半径的一半
double v15;          // [esp+88h] - 高度的一半
```

**关键内存布局**（`var_C8` = `_90` 数组）：
- `_90[9]` (offset +0x48): 底部X半径
- `_90[10]` (offset +0x50): 底部Y半径
- `_90[11]` (offset +0x58): 高度

## 2. 生成流程详解

### 2.1 初始化阶段

```c
// 1. 追踪入口（调试用）
if (unk_117CBD30)
    MTRENT("boxlib/MBXSNO", 13, "%");

// 2. 初始化单位矩阵
MUNIT(&v12);  // v12 是单位变换矩阵

// 3. 计算几何参数（全部乘以 0.5）
v13 = *a4 * 0.5;  // 底部X半径的一半
v14 = *a5 * 0.5;  // 底部Y半径的一半
v15 = *a3 * 0.5;  // 高度的一半

// 4. 存储到底部变换矩阵
_90[9]  = v13;   // 底部X半径
_90[10] = v14;   // 底部Y半径
_90[11] = v15;   // 高度（Z方向）
```

### 2.2 底部圆生成

```c
// 1. 连接变换矩阵
CONCAT(_90, a2, &v12);  // 将单位矩阵与输入变换矩阵连接

// 2. 设置标志位
v9 = -1;  // 0xFFFFFFFF，表示初始化模式

// 3. 准备顶部半径（注意：这里用的是 a6，可能是参数命名问题）
v10 = *a6 * 0.5;  // 顶部X半径的一半

// 4. 调用 MBXCIR 生成底部圆的边界框
MBXCIR(_90, &v10, a8, &v9);
```

**关键操作**：在调用 `MBXCIR` 之前，**没有修改** `_90` 中的位置参数，这意味着底部圆位于：
- X: `v13` (底部X半径)
- Y: `v14` (底部Y半径)
- Z: `v15` (高度的一半，即顶部位置)

**注意**：这里有个逻辑问题，代码使用了 `a6`（顶部半径）来计算底部圆。可能是：
1. 参数传递错误
2. 或者 `a6` 实际上代表底部圆的另一个尺寸参数

### 2.3 顶部圆生成

```c
// 1. 取反底部位置参数（移动到顶部）
_90[9]  = -v13;  // -底部X半径
_90[10] = -v14;  // -底部Y半径
_90[11] = -v15;  // -高度的一半（即底部位置）

// 2. 重新连接变换矩阵
CONCAT(_90, a2, &v12);

// 3. 准备顶部半径
v11 = *a7 * 0.5;  // 顶部Y半径的一半

// 4. 调用 MBXCIR 生成顶部圆的边界框
MBXCIR(_90, &v11, a8, &v9);
```

## 3. MBXCIR 函数分析

### 函数签名
```c
int MBXCIR(int a1, double *a2, int a3, _DWORD *a4)
```

**地址**: `0x108928a8`

### 功能说明

`MBXCIR` 函数计算**圆形几何体在变换空间中的轴对齐包围盒（AABB）**。

### 参数含义

| 参数 | 类型 | 含义 |
|------|------|------|
| `a1` | int | 变换矩阵指针（指向 `_90` 数组） |
| `a2` | double* | 圆的半径 |
| `a3` | int | 输出缓冲区（边界框） |
| `a4` | _DWORD* | 标志位（输入/输出） |

### 实现逻辑

```c
// 遍历 X, Y, Z 三个轴
for (i = 1; i <= 3; ++i) {
    // 1. 获取变换矩阵中第 (i+6) 个元素的平方
    //    这应该是单位方向向量在变换后的值
    double dir_component = *(double *)(a1 + 8 * (i + 6) - 8);
    X = 1.0 - dir_component * dir_component;
    
    // 2. 计算圆在该轴上的投影半径
    if (X >= 0.0)
        v7 = *a2 * sqrt(X);  // 投影半径 = 半径 * sqrt(1 - dir²)
    else
        v7 = 0.0;
    
    // 3. 获取变换后的中心位置
    double center = *(double *)(a1 + 8 * (i + 9) - 8);
    
    // 4. 计算边界
    v8 = center - v7;  // 最小值
    v9 = center + v7;  // 最大值
    
    // 5. 更新或设置边界框
    if ((*a4 & 1) != 0) {
        // 初始化模式：直接设置
        *(double *)(a3 + 8 * i - 8) = v8;
        *(double *)(a3 + 8 * (i + 3) - 8) = v9;
    } else {
        // 累积模式：取最小/最大
        if (v8 < *(double *)(a3 + 8 * i - 8))
            *(double *)(a3 + 8 * i - 8) = v8;
        if (v9 > *(double *)(a3 + 8 * (i + 3) - 8))
            *(double *)(a3 + 8 * (i + 3) - 8) = v9;
    }
}

// 清除标志位
*a4 = 0;
```

### 数学原理

`MBXCIR` 使用以下数学公式计算圆形在轴上的投影：

**投影半径** = `radius * sqrt(1 - dir_component²)`

其中：
- `dir_component` 是单位圆的方向向量在变换后在该轴上的分量
- 如果 `dir_component = 1`（完全对齐），则投影半径为 0
- 如果 `dir_component = 0`（垂直），则投影半径等于原半径

## 4. 关键辅助函数

### 4.1 MUNIT - 单位矩阵初始化

**地址**: `0x10686c50`

```c
int MUNIT(int *matrix)
```

功能：初始化一个 4x4 单位变换矩阵。

### 4.2 CONCAT - 矩阵连接

**地址**: `0x10686d14`

```c
int CONCAT(int *dst, int *src1, int *src2)
```

功能：将两个变换矩阵连接（相乘），结果存储在 `dst`。

在 Snout 生成中：
- `src1`: 输入变换矩阵 (`a2`)
- `src2`: 单位矩阵 (`&v12`)
- `dst`: 结果矩阵 (`_90`)

### 4.3 MTRENT / MTREX - 追踪函数

用于调试和性能追踪：
- `MTRENT`: 函数入口追踪
- `MTREX`: 函数退出追踪

## 5. 完整生成流程总结

```
MBXSNO 函数执行流程：

1. 初始化
   ├─ MTRENT("boxlib/MBXSNO")  [调试追踪]
   └─ MUNIT(&v12)              [创建单位矩阵]

2. 计算底部参数
   ├─ v13 = *a4 * 0.5  [底部X半径]
   ├─ v14 = *a5 * 0.5  [底部Y半径]
   └─ v15 = *a3 * 0.5  [高度]

3. 设置底部变换矩阵
   ├─ _90[9]  = v13
   ├─ _90[10] = v14
   └─ _90[11] = v15

4. 生成底部圆边界框
   ├─ CONCAT(_90, a2, &v12)  [连接变换]
   ├─ v9 = -1                [初始化标志]
   ├─ v10 = *a6 * 0.5        [顶部X半径？]
   └─ MBXCIR(_90, &v10, a8, &v9)  [计算底部圆AABB]

5. 转换到底部位置
   ├─ _90[9]  = -v13  [取反X]
   ├─ _90[10] = -v14  [取反Y]
   └─ _90[11] = -v15  [取反Z，移动到底部]

6. 生成顶部圆边界框
   ├─ CONCAT(_90, a2, &v12)  [重新连接变换]
   ├─ v11 = *a7 * 0.5        [顶部Y半径]
   └─ MBXCIR(_90, &v11, a8, &v9)  [计算顶部圆AABB]

7. 清理
   └─ MTREX("%")  [调试追踪退出]
```

## 6. 关键发现

### 6.1 边界框计算而非 Mesh 生成

**重要**：`MBXSNO` 函数**不直接生成 mesh**，而是：
1. 计算 Snout 的两个端面（底部和顶部圆）的边界框
2. 通过 `MBXCIR` 计算每个圆在变换空间中的 AABB
3. 将结果累积到输出缓冲区 `a8` 中

### 6.2 参数传递的疑问

代码中有一个**潜在的参数混淆**：
- 第一次调用 `MBXCIR` 使用 `a6`（应该是顶部X半径）
- 第二次调用 `MBXCIR` 使用 `a7`（顶部Y半径）

这可能意味着：
1. 代码有 bug
2. 或者参数含义与预期不同
3. 或者 Snout 支持椭圆柱形，需要分别指定 X/Y 半径

### 6.3 变换矩阵的使用

函数使用变换矩阵 (`a2`) 来：
1. 支持任意方向的 Snout（不仅限于 Z 轴方向）
2. 支持任意位置和旋转
3. 通过 `CONCAT` 组合多个变换

### 6.4 高度参数的使用

高度参数 (`a3`) 被分成两半：
- 第一次：`+height/2` → 顶部位置
- 第二次：`-height/2` → 底部位置

这符合标准的坐标系统，其中 Snout 的中心在原点。

## 7. 与 RVM 格式的对应关系

根据代码库中的实现，RVM 格式的 Snout 参数为：

```rust
Snout([f32; 9])  // [bottom_radius, top_radius, height, offset_x, offset_y, bshear_x, bshear_y, tshear_x, tshear_y]
```

IDA Pro 中的参数映射推测：

| RVM 参数 | IDA Pro 参数 | 说明 |
|----------|--------------|------|
| `bottom_radius` | `*a4` 或 `*a5` | 底部半径 |
| `top_radius` | `*a6` 或 `*a7` | 顶部半径 |
| `height` | `*a3` | 高度 |
| `offset_x/y` | 可能在 `a2` 变换矩阵中 | 偏移量 |
| `bshear/t shear` | 未在此函数中处理 | 剪切角（可能在更高层处理） |

## 8. 总结

### 核心功能

`MBXSNO` 函数的主要目的是：
1. **计算 Snout 的边界框**（AABB）
2. **支持任意变换**（位置、旋转、缩放）
3. **处理椭圆柱形**（不同的 X/Y 半径）

### 设计特点

1. **分层设计**：
   - 底层：边界框计算（`MBXSNO`/`MBXCIR`）
   - 高层：实际 mesh 生成（在其他模块中）

2. **优化策略**：
   - 使用边界框进行快速剔除
   - 分离几何计算和渲染逻辑

3. **灵活性**：
   - 支持任意方向的 Snout
   - 支持椭圆柱形截面

### 实际 Mesh 生成

实际的 mesh 生成（顶点、法线、三角形）**不在 `MBXSNO` 函数中**，而是在更高层的渲染管道中完成。`MBXSNO` 只为渲染器提供：
- 边界框信息（用于剔除和空间索引）
- 几何参数（用于后续的 mesh 生成）
