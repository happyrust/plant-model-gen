# IDA Pro 记录：`NOUN_SCYL`

## 快照
- 位于 `0x10a698e4` 的字符串字面值 `aScyl` 保存了四字节的名词代码 "SCYL"，它会被传入与其他 `CYL*` 名词共用的构造辅助函数。
- 全局指针 `?NOUN_SCYL@@3QBVDB_Noun@@B` 存放在 `0x10f5f3e0`。在专用初始化函数执行之前，它的内容是 `0xffffffff`。
- `sub_1091A100`（0x1091A100）负责分配对象，并用临时缓冲中的 "SCYL" 调用 `DB_Noun::DB_Noun`，然后把返回的对象指针写入 `NOUN_SCYL`。清理流程与其他名词构造器（例如负责 `NOUN_CYLI` 的 `sub_108F0AC0`）保持一致。

## 初始化流程
- 该构造辅助函数遵循通用模板：
  1. 通过 `operator new` 分配 0x164 字节。
  2. 调用 `sub_10004A90` 构造短小的 `std::basic_string`，内容为名词助记符。
  3. 调用 `DB_Noun::DB_Noun` 构造运行时描述对象。
  4. 将得到的指针保存到 `NOUN_SCYL`，若临时字符串容量超过 SSO 阈值则释放其缓冲区。
- 位于 `0x10a17288` 的跳转表保存着 `sub_1091A100` 以及相邻的名词初始化函数（`sub_1091A1D0`、`sub_1091A2A0` 等）。DLL 启动阶段会遍历该表，逐一构造所有 `NOUN_*` 单例。

## 注册表结构
- 在名词指针块之前存在一个位于 `0x10af7de0` 的结构体，它指向 `0x10af7df8`。后者保存着先前内存转储中看到的长串 DWORD 值。沿着该结构遍历可以发现，这是所有名词单例共用的元数据表（每个条目都会指向与 `NOUN_CYLI`、`NOUN_PCYL`、`NOUN_LCYL` 等相同的内存区域）。
- 由于 `NOUN_SCYL` 是通过这个通用注册表而不是直接符号引用访问的，IDA 只展示了两个交叉引用：构造器赋值以及元数据表。目前 `core.dll` 中没有独立例程直接读取该全局变量。

## 与圆柱工具的关系
- `sub_104E22B0`、`sub_104EEBB0`、`sub_104E3EA0` 等大型检查辅助函数展示了圆柱类名词的典型使用方式：通过 `NOUN_CYLI` 实例化 `DB_PredicateType`，在 `ATT_MEMB` 等限定符下遍历成员，并打印诸如 “Cylinder Members = …” 的诊断信息。
- 这些例程并未触及 `NOUN_SCYL`，而是集中在 `NOUN_CYLI`（以及相关的 `FLAN`、`PIPE` 名词）上遍历装配层级。这说明 `SCYL` 要么保留给其他模块使用，要么预期通过名词注册表间接解析。

## 未解问题 / 后续工作
- 确认套件中的其他二进制是否通过共享注册表使用 `NOUN_SCYL`（例如跟踪对 `DB_Noun::fullName` 或构造函数内部填充的哈希表的查询）。
- 在初始化完成后捕获一次运行时快照，检查 `DB_Noun` 对象内容，尤其是其他圆柱例程用于限定符选择的 `base_data[112]` 附近偏移。
- 如果期望 `SCYL` 与 `CYLI` 扮演相同角色，可以考虑在现有的详尽转储工具中加入它，并验证是否存在面向 “S Cylinder” 变体的额外属性。
