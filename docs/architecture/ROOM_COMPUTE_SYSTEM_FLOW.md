# 房间计算系统完整流程架构

## 系统概览

房间计算系统是一个异步任务驱动的计算引擎，包含四个核心子系统：

- **任务管理系统**：API请求接收、任务调度、状态管理
- **关系重建系统**：房间-面板-构件关系计算
- **空间查询系统**：基于混合索引的几何空间查询
- **数据迁移系统**：版本控制与数据迁移

## 六、关键计算流程总结

### 6.1 房间关系构建计算流程

1. **输入**：房间关键词列表
2. **步骤**：
   - 查询匹配房间关键词的面板 → `room_panel_map`
   - 并发处理每个房间面板（并发度=4）
   - 对每个面板：
     - 加载几何网格
     - 空间索引查询候选构件
     - 几何精确检测 → `Vec<RefnoEnum>`
   - 批量保存关系到 SurrealDB
3. **输出**：房间-面板-构件关系网络

### 6.2 点查询房间计算流程

1. **输入**：空间坐标点 (x, y, z)
2. **步骤**：
   - 混合索引粗筛 → 候选面板列表
   - 加载候选面板几何网格
   - 精确点包含检测 → 匹配的面板
   - 查询面板对应的房间号
3. **输出**：`(room_number, panel_refno)`

### 6.3 数据迁移计算流程

1. **输入**：迁移配置
2. **步骤**：
   - 创建迁移前快照 (版本控制)
   - 执行数据结构转换和迁移
   - 创建迁移后快照
   - 验证数据完整性
3. **输出**：迁移报告和验证结果

## 七、性能优化要点

### 7.1 并发处理

- 使用 `buffer_unordered(4)` 控制并发度
- 避免过度并发导致内存压力

### 7.2 空间索引优化

- 混合索引策略：内存R-Tree + SQLite空间索引
- 两阶段过滤：粗筛 + 精确检测

### 7.3 批量操作

- 批量构建 SurrealDB RELATE 语句
- 减少数据库往返次数

### 7.4 异步架构

- tokio 异步运行时
- 非阻塞任务调度
- WebSocket 实时进度推送
