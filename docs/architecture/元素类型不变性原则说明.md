# 元素类型不变性原则说明

## 核心原则

在 PDMS/E3D 系统中，**元素类型（Element Type）一旦创建就不会改变**。这是系统的基本设计原则。

## 重要影响

### 1. 简化查询逻辑
由于元素类型不变，我们不需要：
- `get_element_type_at_sesno(refno, sesno)` - 不需要
- 在历史查询中考虑类型变化 - 不需要
- 在 element_changes 表中记录类型变更 - 不需要

只需要：
- `get_element_type(refno)` - 获取当前类型即可

### 2. 优化数据结构
```rust
// 增量记录中，element_type 只需要记录一次
pub struct ElementChange {
    pub refno: String,
    pub sesno: u32,
    pub element_type: String, // 固定值，不会变化
    // ... 其他字段
}
```

### 3. 简化增量计算
```rust
// 处理增量时，可以直接根据当前类型分类
async fn classify_affected_elements(
    affected_refnos: HashSet<RefnoEnum>,
) -> (Vec<RefnoEnum>, Vec<RefnoEnum>, Vec<RefnoEnum>) {
    let mut prim_refnos = Vec::new();
    let mut loop_refnos = Vec::new();
    let mut bran_refnos = Vec::new();
    
    for refno in affected_refnos {
        // 直接查询当前类型，无需考虑历史
        let type_name = get_type_name(refno).await?;
        match type_name.as_str() {
            "PRIM" => prim_refnos.push(refno),
            "LOOP" => loop_refnos.push(refno),
            "BRAN" | "HANGER" => bran_refnos.push(refno),
            _ => {}
        }
    }
    
    (prim_refnos, loop_refnos, bran_refnos)
}
```

## 可能变化的属性

虽然类型不变，但以下内容可能随时间变化：
- 几何数据（位置、尺寸、形状）
- 属性值（名称、描述、参数）
- 层级关系（父子关系、引用关系）
- 存在性（创建、删除）

## 实际应用

### 1. 在 Raphtory 中
```rust
// 节点创建时设置类型，后续不再改变
graph.add_node(
    refno.to_string(),
    sesno as i64,
    vec![
        ("type", element_type), // 一次性设置，永不改变
        ("created_at", sesno.to_string()),
    ],
)?;
```

### 2. 在查询优化中
```rust
// 可以在内存中缓存 refno -> type 的映射
lazy_static! {
    static ref ELEMENT_TYPE_CACHE: DashMap<RefnoEnum, String> = DashMap::new();
}

pub async fn get_element_type_cached(refno: RefnoEnum) -> String {
    if let Some(cached) = ELEMENT_TYPE_CACHE.get(&refno) {
        return cached.clone();
    }
    
    let type_name = get_type_name(refno).await.unwrap();
    ELEMENT_TYPE_CACHE.insert(refno, type_name.clone());
    type_name
}
```

## 总结

元素类型不变性是系统的基础约束，理解并利用这一原则可以：
1. 简化代码逻辑
2. 提升查询性能
3. 减少存储开销
4. 避免不必要的复杂性

在设计和实现时，应始终记住：**类型是元素的固有属性，不是时变属性**。
