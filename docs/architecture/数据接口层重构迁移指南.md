# 数据接口层重构迁移指南

## 概述

本次重构将 `gen_geos` 模块从直接调用 `aios_core` 函数改为通过数据接口层（`PdmsDataInterface`）访问数据，实现了数据访问的解耦和标准化。

## 主要改动

### 更新：模型生成函数已全部加入接口（2024年最新）

最新的改动将所有模型生成函数都加入到了 `PdmsDataInterface` 接口中，包括：
- `gen_all_geos_data` - 生成所有几何体数据
- `gen_geos_data_by_dbnum` - 按数据库编号生成
- `gen_geos_data` - 核心生成函数
- `process_meshes_by_dbnos` - 处理网格数据
- `query_tubi_size` - 查询管道尺寸

### 1. 扩展了 PdmsDataInterface 接口

在 `src/data_interface/interface.rs` 中添加了以下新接口：

```rust
// 查询接口
async fn query_type_refnos_by_dbnum(...) -> Result<Vec<RefnoEnum>>;
async fn query_use_cate_refnos_by_dbnum(...) -> Result<Vec<RefnoEnum>>;
async fn query_mdb_db_nums(...) -> Result<Vec<u32>>;
async fn query_multi_children_refnos(...) -> Result<Vec<RefnoEnum>>;
async fn query_multi_deep_versioned_children_filter_inst(...) -> Result<HashSet<RefnoEnum>>;
async fn query_multi_deep_children_filter_spre(...) -> Result<HashSet<RefnoEnum>>;
async fn query_group_by_cata_hash(...) -> Result<DashMap<String, CataHashRefnoKV>>;

// 数据访问接口
async fn get_children_pes(...) -> Result<Vec<Pe>>;
async fn get_pe(...) -> Result<Option<Pe>>;
async fn get_named_attmap(...) -> Result<NamedAttrMap>;

// 执行接口
async fn execute_sql<T>(...) -> Result<Response>;
async fn save_instance_data(...) -> Result<()>;
async fn gen_meshes_in_db(...) -> Result<()>;
async fn booleans_meshes_in_db(...) -> Result<()>;
```

### 2. 实现了接口方法

在 `src/data_interface/tidb_manager.rs` 的 `AiosDBManager` 中实现了所有新增接口。

### 3. 重构了 gen_model.rs

创建了 `src/fast_model/gen_model_refactored.rs`，主要改动：

- 所有函数添加了 `data_interface: Arc<dyn PdmsDataInterface>` 参数
- 将直接调用改为通过接口调用：
  ```rust
  // 旧代码
  aios_core::query_type_refnos_by_dbnum(&["ZONE"], dbno, Some(true), gen_history).await?
  
  // 新代码
  data_interface.query_type_refnos_by_dbnum(&["ZONE"], dbno, Some(true), gen_history).await?
  ```

### 4. 创建了工厂类和依赖注入

新增 `src/data_interface/database_factory.rs`：
- `DatabaseFactory`: 创建数据接口实例
- `DiContainer`: 依赖注入容器
- 支持多种数据库类型（TiDB、SurrealDB、混合模式）

## 迁移步骤

### 1. 创建数据接口实例（推荐使用最新方式）

```rust
// 方式1: 使用默认配置
let data_interface = DatabaseFactory::create_default().await?;

// 方式2: 使用自定义配置
let config = DatabaseConfig {
    db_type: DatabaseType::TiDB,
    db_option: DbOption::from_args(),
    project_path: "/path/to/project".to_string(),
    projects: vec!["project1".to_string()],
};
let data_interface = DatabaseFactory::create_interface(config).await?;
```

### 2. 使用新的接口调用方式（推荐）

#### 旧方式（需要导入函数并传递接口）：
```rust
use gen_model::fast_model::gen_model_refactored::gen_all_geos_data;

let result = gen_all_geos_data(
    manual_refnos,
    &db_option,
    incr_updates,
    data_interface.clone(),  // 需要传递接口
).await?;
```

#### 新方式（直接通过接口调用）：
```rust
// 不需要导入函数，直接调用
let result = data_interface.gen_all_geos_data(
    manual_refnos,
    &db_option,
    incr_updates,
).await?;

// 其他函数也类似
let db_model_refnos = data_interface.gen_geos_data_by_dbnum(
    dbno,
    db_option_arc,
    sender,
).await?;
```

### 3. 替换直接调用

将所有直接调用 `aios_core` 的地方改为通过数据接口：

```rust
// 查询类
aios_core::query_xxx() -> data_interface.query_xxx()

// 获取类  
aios_core::get_xxx() -> data_interface.get_xxx()

// SQL 查询
SUL_DB.query() -> data_interface.execute_sql()
```

### 4. 使用依赖注入（可选）

对于大型应用，建议使用依赖注入模式：

```rust
// 创建容器
let container = DiContainer::new(data_interface);

// 在服务中使用
struct MyService {
    container: DiContainer,
}

impl MyService {
    async fn do_work(&self) {
        let data_interface = self.container.data_interface();
        // 使用 data_interface ...
    }
}
```

## 优势

1. **解耦**: 业务逻辑与数据访问分离
2. **可测试**: 便于创建 Mock 实现进行单元测试
3. **可扩展**: 轻松切换不同的数据源实现
4. **统一管理**: 所有数据访问通过统一接口
5. **简化调用**: 不需要导入具体函数，直接通过接口调用
6. **更好的封装**: 实现细节完全隐藏在接口后面

## 注意事项

1. 确保所有新增的查询都通过接口层
2. 避免在业务代码中直接使用 `aios_core` 或全局实例
3. 合理使用批量接口减少数据库访问次数
4. 考虑使用连接池和缓存优化性能
