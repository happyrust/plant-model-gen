# 部署站点功能企划文档

## 1. 项目背景

### 1.1 现状分析
- 当前系统支持单个项目的管理和部署
- 用户需求：从指定目录扫描多个E3D项目，创建包含多个E3D项目的部署站点
- 原"新增项目"功能需重构为"新增部署站点"

### 1.2 核心需求
1. 扫描指定目录下的所有E3D项目
2. 用户可选择多个E3D项目创建部署站点
3. 一个部署站点可包含多个E3D项目
4. 统一管理和配置多个项目的部署

## 2. 功能设计

### 2.1 目录扫描功能
**已有基础**：`DirectoryScanRequest/DirectoryScanResult`

**核心流程**：
1. 用户指定根目录路径
2. 系统递归扫描E3D项目文件
3. 识别项目信息：名称、路径、数据库文件数、大小等
4. 返回项目列表供用户选择

**技术实现**：
```rust
// 已存在的结构体，无需修改
pub struct DirectoryScanRequest {
    pub directory_path: String,
    pub recursive: bool,
    pub max_depth: Option<u32>,
}

pub struct DirectoryScanResult {
    pub root_directory: String,
    pub projects: Vec<ProjectInfo>,
    pub scan_duration_ms: u64,
    pub scanned_directories: u32,
    pub errors: Vec<String>,
}
```

### 2.2 部署站点模型设计

**新增数据结构**：
```rust
/// E3D项目信息
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct E3dProjectInfo {
    pub name: String,
    pub path: String,
    pub project_code: Option<u32>,
    pub db_file_count: u32,
    pub size_bytes: u64,
    pub selected: bool,  // 是否被选中
}

/// 部署站点（替代原ProjectItem）
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DeploymentSite {
    pub id: Option<String>,
    pub name: String,
    pub description: Option<String>,
    pub e3d_projects: Vec<E3dProjectInfo>,  // 包含的E3D项目列表
    pub config: DatabaseConfig,
    pub status: ProjectStatus,
    pub url: Option<String>,
    pub env: Option<String>,
    pub owner: Option<String>,
    pub created_at: Option<String>,
    pub updated_at: Option<String>,
}

/// 创建部署站点请求
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DeploymentSiteCreateRequest {
    pub name: String,
    pub description: Option<String>,
    pub root_directory: String,
    pub selected_projects: Vec<String>,  // 选中的项目路径列表
    pub config: DatabaseConfig,
    pub env: Option<String>,
    pub owner: Option<String>,
}
```

### 2.3 API接口设计

**新增接口**：
```rust
// 1. 扫描目录API（已存在，可复用）
POST /api/scan-directory
Body: DirectoryScanRequest
Response: DirectoryScanResult

// 2. 创建部署站点API  
POST /api/deployment-sites
Body: DeploymentSiteCreateRequest
Response: DeploymentSite

// 3. 获取部署站点列表API
GET /api/deployment-sites
Query: ProjectQuery
Response: Vec<DeploymentSite>

// 4. 更新部署站点API
PUT /api/deployment-sites/{id}
Body: DeploymentSiteUpdateRequest
Response: DeploymentSite

// 5. 删除部署站点API
DELETE /api/deployment-sites/{id}
Response: 200 OK
```

## 4. 技术风险与解决方案

### 4.1 数据迁移风险
**风险**：现有项目数据需要迁移到新的部署站点模型
**解决方案**：
- 编写数据迁移脚本
- 支持向后兼容
- 提供回滚机制

### 4.2 性能风险
**风险**：大目录扫描可能耗时较长
**解决方案**：
- 异步扫描处理
- 进度反馈机制
- 扫描结果缓存

### 4.3 存储风险
**风险**：多项目站点可能占用大量存储
**解决方案**：
- 软链接而非复制项目文件
- 引用式存储项目路径
- 定期清理无效站点
