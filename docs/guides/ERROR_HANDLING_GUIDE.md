# AIOS 数据库管理平台 - 错误处理指南

## 🎯 概述

AIOS 数据库管理平台现在具备了完善的错误处理和日志系统，当任务执行失败时，您可以查看详细的错误信息、原因分析和解决方案建议。

## 🔍 错误查看方式

### 1. 任务列表中的错误指示
- **状态显示**：失败的任务会显示红色的"失败"状态
- **错误按钮**：失败任务的操作列会显示橙色的警告图标 ⚠️
- **点击查看**：点击警告图标可查看详细错误信息

### 2. 错误详情模态框
点击错误按钮后，会弹出详细的错误信息窗口，包含：

#### 📋 错误概要
- **任务名称**：失败的任务名称
- **错误代码**：系统分配的错误代码（便于技术支持）
- **失败步骤**：具体在哪个步骤失败
- **错误消息**：详细的错误描述

#### 💡 建议解决方案
- **智能分析**：系统根据错误类型自动提供解决方案
- **分步指导**：具体的操作步骤和检查项目
- **优先级排序**：按照可能性排序的解决方案

#### ⚙️ 相关配置信息
- **配置参数**：导致错误的相关配置信息
- **JSON格式**：便于技术人员分析的结构化数据

#### 📝 错误日志
- **时间戳**：精确的错误发生时间
- **日志级别**：Error（错误）或Critical（严重错误）
- **详细消息**：完整的错误日志信息

#### 🔧 堆栈跟踪
- **技术详情**：供开发人员分析的堆栈信息
- **调试信息**：帮助定位问题根源

## 🚨 常见错误类型及解决方案

### 1. 数据库连接错误 (DB_CONN_001)

**错误表现**：
- 任务在"初始化数据库连接"步骤失败
- 错误消息包含"connection"或"database"

**可能原因**：
- SurrealDB服务未启动
- 网络连接问题
- 数据库配置错误
- 用户权限不足

**解决方案**：
1. 检查SurrealDB服务是否正在运行
2. 验证DbOption.toml配置文件中的数据库连接参数
3. 确认网络连接和防火墙设置
4. 检查数据库用户权限

### 2. 几何数据生成错误

#### 内存不足 (GEO_MEM_001)
**错误表现**：
- 错误消息包含"memory"或"allocation"

**解决方案**：
1. 增加系统可用内存
2. 减少批处理大小
3. 关闭其他占用内存的程序
4. 检查是否有内存泄漏

#### 网格生成错误 (GEO_MESH_001)
**错误表现**：
- 错误消息包含"mesh"或"geometry"

**解决方案**：
1. 检查几何数据的完整性
2. 调整网格容差参数
3. 验证输入数据格式
4. 检查OCC几何库配置

#### 权限错误 (GEO_PERM_001)
**错误表现**：
- 错误消息包含"permission"或"access"

**解决方案**：
1. 检查文件系统权限
2. 确认assets/meshes目录可写
3. 验证数据库写入权限

### 3. 空间树生成错误

#### AABB树错误 (SPATIAL_TREE_001)
**错误表现**：
- 错误消息包含"aabb"或"tree"

**解决方案**：
1. 检查AABB树文件是否损坏
2. 尝试重新构建空间索引
3. 验证几何数据的完整性
4. 检查空间树配置参数

#### 房间关系错误 (SPATIAL_ROOM_001)
**错误表现**：
- 错误消息包含"room"或"panel"

**解决方案**：
1. 检查房间关键字配置
2. 验证房间和面板数据
3. 确认空间关系计算参数
4. 检查项目特定的房间匹配规则

## 🛠️ 错误处理最佳实践

### 1. 预防性检查
- **启动前验证**：服务器启动时会自动验证数据库连接
- **配置检查**：确保DbOption.toml配置正确
- **资源监控**：定期检查系统资源使用情况

### 2. 错误发生时的处理步骤
1. **查看错误详情**：点击任务列表中的错误按钮
2. **分析错误类型**：根据错误代码和消息判断问题类型
3. **按建议操作**：按照系统提供的解决方案逐步排查
4. **记录问题**：保存错误信息供后续分析

### 3. 联系技术支持
如果按照建议解决方案仍无法解决问题，请提供：
- **错误代码**：如GEO_MEM_001
- **完整错误消息**：从错误详情中复制
- **相关配置**：任务配置和系统配置
- **堆栈跟踪**：技术详情信息

## 📊 错误统计和分析

### API接口
- `GET /api/tasks/:id/error` - 获取特定任务的错误详情
- 返回完整的错误信息，包括建议解决方案

### 日志级别
- **Debug**：调试信息
- **Info**：一般信息
- **Warning**：警告信息
- **Error**：错误信息
- **Critical**：严重错误

## 🔄 错误恢复

### 自动重试
- 某些临时性错误（如网络超时）可以通过重新启动任务解决
- 系统会记录重试次数和结果

### 手动干预
- 配置错误需要修改配置后重新执行
- 权限问题需要系统管理员处理
- 资源不足需要优化系统配置
