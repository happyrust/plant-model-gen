# 世界矩阵计算函数迁移指南

## 概述

本文档指导如何从旧的 `get_world_mat4` 函数迁移到新的 `get_world_mat4_with_strategies` 函数，后者使用重构后的策略模式系统。

## 重构收益

### 新函数的优势
- **策略模式支持**：为不同构件类型（GENSEC、SJOI、ENDATU等）提供专门的计算策略
- **更好的可维护性**：每种类型的逻辑独立封装，易于修改和扩展
- **数学鲁棒性**：修复了 `cal_ori_by_ydir` 的共线问题
- **类型安全**：利用 Rust 类型系统保证正确性
- **测试友好**：每个策略可独立测试

### 性能特性
- **缓存优化**：新旧函数都使用 `#[cached]` 属性，性能相当
- **异步支持**：充分利用 Rust async 特性
- **错误处理**：更好的错误传播和调试信息

### Q: 新函数会产生不同的结果吗？
A: 在大多数情况下结果应该完全相同。细微差异可能来自：
- 浮点数计算顺序的改变

## 代码示例

### 基本使用
```rust
use crate::{RefnoEnum, rs_surreal::spatial::get_world_mat4_with_strategies};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let refno: RefnoEnum = "17496/266212".parse()?;
    
    // 计算世界变换
    let world_transform = get_world_mat4_with_strategies(refno, false).await?;
    
    // 计算局部变换
    let local_transform = get_world_mat4_with_strategies(refno, true).await?;
    
    println!("世界变换: {:?}", world_transform);
    println!("局部变换: {:?}", local_transform);
    
    Ok(())
}
```

### 错误处理
```rust
match get_world_mat4_with_strategies(refno, false).await {
    Ok(Some(matrix)) => {
        println!("变换计算成功: {:?}", matrix);
    }
    Ok(None) => {
        println!("无法计算变换（可能缺少必要属性）");
    }
    Err(e) => {
        eprintln!("变换计算失败: {}", e);
    }
}
```

|------|--------|-------------------|------|

### 性能特性

#### 共同优势
- **缓存机制**：两个实现都使用 `#[cached]` 属性，缓存命中时性能相同
- **异步优化**：充分利用 Rust async 特性
- **内存管理**：使用 `glam` 高性能数学库

#### 新实现优势
- **策略开销**：策略分派的额外开销 < 1ms
- **错误处理**：更好的错误传播，减少重试次数
- **数学精度**：修复的数学算法可能减少迭代次数

#### 性能建议
- 在高频调用场景中，缓存命中时性能差异可忽略
- 首次计算的微小差异（~2ms）在大多数应用中可接受
- 建议在生产环境中启用性能监控以验证实际表现
