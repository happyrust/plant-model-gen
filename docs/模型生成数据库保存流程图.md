# 模型生成数据库保存流程图

## 快速概览

```
生成模型 (gen_model) 数据保存流程图
==========================================

输入: 原始元素 (pe:{refno})
     ↓
[阶段1: 几何体生成]
     ↓
生成 ShapeInstancesData
     ├─ inst_info_map {refno -> InstInfo}
     ├─ inst_geos_map {inst_key -> InstGeosData}
     ├─ inst_tubi_map {refno -> TubiInfo}
     ├─ neg_relate_map {主元素 -> 被减元素列表}
     └─ ngmr_neg_relate_map
     ↓
save_instance_data() 并行插入以下数据:
     ↓
┌─────────────────────────────────────┐
│ 数据库表:                           │
│ 1. inst_info    (实例信息)          │
│ 2. inst_geo     (几何体)            │
│ 3. trans        (变换矩阵)          │
│ 4. aabb         (包围盒)            │
│ 5. vec3         (关键点)            │
│ 6. geo_relate   (几何关系)          │
│ 7. inst_relate  (实例关系)          │
│ 8. neg_relate   (负关系)            │
│ 9. ngmr_relate  (负GM关系)         │
└─────────────────────────────────────┘
     ↓
[阶段2: 网格生成]
     ↓
gen_meshes_in_db()
     ├─ 读取几何体数据
     ├─ OpenCascade 生成三角网格
     ├─ 保存到 assets/meshes/{geo_hash}.mesh
     └─ 更新 inst_geo.meshed = true
     ↓
[阶段3: 布尔运算]
     ↓
booleans_meshes_in_db()
     ├─ 读取正实体和负实体网格
     ├─ Manifold 执行布尔减法
     ├─ 保存结果网格
     └─ 创建新的 inst_geo 记录
     ↓
输出: 完整的3D模型数据
```

## 数据关系图

### 核心关系链

```
pe:{refno} (原始元素)
    │
    ├─[inst_relate: inst_relate:{refno}]
    │                    │
    │                    ├─> world_trans: trans:⟨hash⟩
    │                    ├─> zone_refno: {refno}
    │                    ├─> dt: timestamp
    │                    └─> out: inst_info:⟨id⟩
    │
    └─[inst_info: inst_info:⟨id⟩]
                          │
                          ├─> [geo_relate]
                          │         │
                          │         ├─> out: inst_geo:⟨geo_hash⟩
                          │         ├─> trans: trans:⟨hash⟩
                          │         ├─> pts: [vec3:⟨hash⟩, ...]
                          │         └─> cata_neg: [pe:{refno}, ...]
                          │
                          ├─> [neg_relate] (如果有负关系)
                          │
                          └─> [ngmr_relate] (如果有负GM)
```

### 数据流向

```
1. 元素查询
   pe:{refno}
     ↓
   
2. 几何参数计算
   ├─> geo_param (几何参数)
   ├─> transform (变换矩阵)
   └─> aabb (包围盒)
     ↓
   
3. 数据库保存 (并行)
   ├─> trans:⟨hash⟩ {d: Transform}
   ├─> aabb:⟨hash⟩ {d: Aabb}
   ├─> vec3:⟨hash⟩ {d: Vec3} × N
   ├─> inst_info:⟨id⟩ {...}
   ├─> inst_geo:⟨geo_hash⟩ {meshed: false, ...}
   ├─> inst_relate:{refno} {in: pe, out: inst_info, ...}
   ├─> geo_relate {in: inst_info, out: inst_geo, ...}
   ├─> neg_relate {in: pe, out: pe, ...} (可选)
   └─> ngmr_relate {in: pe, out: pe, ngmr: pe} (可选)
     ↓
   
4. 网格文件生成
   ┌─────────────────────────────────┐
   │ assets/meshes/{geo_hash}.mesh  │
   │ - 顶点数据 (vertices)            │
   │ - 索引数据 (indices)             │
   │ - 包围盒 (aabb)                  │
   └─────────────────────────────────┘
     ↓
   
5. 数据库更新
   UPDATE inst_geo:⟨geo_hash⟩ SET meshed = true, aabb = aabb:⟨hash⟩
   UPSERT inst_relate_aabb:⟨refno⟩ SET refno = pe:⟨refno⟩, aabb = aabb:⟨hash⟩, updated_at = time::now()
     ↓

6. 布尔运算 (如果有负实体)
   ┌─────────────────────────────┐
   │ 加载正实体和负实体网格       │
   │ ├─> 正实体: {geo_hash}_p   │
   │ └─> 负实体: {geo_hash}_n   │
   └─────────────────────────────┘
     ↓
   ┌─────────────────────────────┐
   │ Manifold 布尔减法            │
   │ Result = Pos - Neg1 - Neg2.. │
   └─────────────────────────────┘
     ↓
   保存结果网格: {hash_with_neg}.mesh
     ↓
   CREATE inst_geo:⟨result_hash⟩
   CREATE geo_relate → inst_geo:⟨result_hash⟩
```

## 数据类型说明

### 1. 原始输入数据

```rust
// 元素引用号
RefnoEnum { dbno, noun, id }

// 几何体实例数据
ShapeInstancesData {
    inst_info_map: HashMap<RefnoEnum, InstInfo>,
    inst_geos_map: HashMap<String, InstGeosData>,
    inst_tubi_map: HashMap<RefnoEnum, TubiInfo>,
    neg_relate_map: HashMap<RefnoEnum, Vec<RefnoEnum>>,
    ngmr_neg_relate_map: HashMap<RefnoEnum, Vec<(RefnoEnum, RefnoEnum)>>,
}
```

### 2. 数据库记录结构

#### inst_relate (关系表)

```sql
{
    id: inst_relate:{refno},
    in: pe:{refno},
    out: inst_info:⟨uuid⟩,
    world_trans: trans:⟨hash⟩,
    generic: 'EQUI' | 'PIPE' | 'ELBO' | ...,
    zone_refno: pe:{zone_refno},  -- 自动填充
    dt: timestamp,                 -- 自动填充
    has_cata_neg: boolean,
    solid: boolean
}
```

#### geo_relate (关系表)

```sql
{
    id: {hash},
    in: inst_info:⟨uuid⟩,
    out: inst_geo:⟨geo_hash⟩,
    trans: trans:⟨hash⟩,
    geom_refno: pe:{refno},
    pts: [vec3:⟨hash⟩, vec3:⟨hash⟩, ...],
    geo_type: 'Pos' | 'Neg',
    visible: boolean,
    cata_neg: [pe:{refno}, ...]  -- 可选
}
```

#### inst_geo (节点表)

```sql
{
    id: inst_geo:⟨geo_hash⟩,
    meshed: boolean,
    aabb: aabb:⟨hash⟩,  -- 可选项
    param: {...}         -- 几何参数
}
```

#### trans (键值对表)

```sql
{
    id: trans:⟨hash⟩,
    d: "{...JSON序列化的Transform...}"
}
```

#### aabb (键值对表)

```sql
{
    id: aabb:⟨hash⟩,
    d: "{...JSON序列化的Aabb...}"
}
```

#### vec3 (键值对表)

```sql
{
    id: vec3:⟨hash⟩,
    d: "{...JSON序列化的Vec3...}"
}
```

## 批次处理策略

### 批量大小
- 默认: 300 条记录/批
- 使用 `chunks(chunk_size)` 分批处理

### 并发处理
```rust
let mut db_futures = FuturesUnordered::new();

// 为每个批次创建异步任务
db_futures.push(tokio::spawn(async move {
    db.query(sql).await
}));

// 等待所有任务完成
while let Some(result) = db_futures.next().await {
    // 处理结果
}
```

## 性能优化措施

### 1. 数据去重
```rust
// 使用 HashMap 避免重复保存相同的变换矩阵
if !transform_map.contains_key(&transform_hash) {
    transform_map.insert(transform_hash, json_string);
}
```

### 2. 预插入单位矩阵
```rust
// 减少后续查询和计算
transform_map.insert(0, IDENTITY_TRANSFORM_JSON);
```

### 3. 并行插入
```rust
// 不同的表可以并行插入
- inst_geo 和 geo_relate 可以并行
- inst_info 和 trans/aabb/vec3 可以并行
- 关系表需要等待节点表插入完成
```

## 错误处理

### 1. NaN 值过滤
```rust
if transform.translation.is_nan() 
    || transform.rotation.is_nan() 
    || transform.scale.is_nan() 
{
    continue; // 跳过该数据
}
```

### 2. 级联删除
```rust
if replace_exist {
    delete_inst_relate_cascade(&refnos, chunk_size).await?;
}
```

### 3. 事务处理
```rust
// INSERT IGNORE 避免重复插入错误
"insert ignore into {table} [{json_array}]";
```

## 数据量统计

### 典型场景
- 小型模型: 100-1,000 个元素
- 中型模型: 1,000-10,000 个元素  
- 大型模型: 10,000-100,000 个元素

### 数据表大小估算 (按元素)
- `inst_relate`: 1 条/元素
- `inst_info`: 1 条/元素
- `inst_geo`: 1-N 条/元素（几何体复用）
- `geo_relate`: 1-N 条/元素
- `trans`: 1-N 条（去重后）
- `aabb`: 1-N 条（去重后）
- `vec3`: N 条（关键点）

### 网格文件大小
- 平均: 10-100 KB/几何体
- 大型设备: 1-10 MB

## 关键函数调用链

```
主入口
  └─> gen_all_geos_data()
       └─> gen_geos_data() / gen_geos_data_by_dbnum()
            ├─> gen_cata_geos()      (元件库几何)
            ├─> gen_loop_geos()      (回路几何)
            ├─> gen_prim_geos()      (基本体几何)
            └─> save_instance_data()  ← ★ 核心保存函数
                 ├─> 删除旧数据 (可选)
                 ├─> 并发插入所有表
                 └─> 更新 pe 表 has_inst 字段
                 
       (如果 gen_mesh=true)
       └─> process_meshes_update_db_deep()
            └─> gen_meshes_in_db()
                 ├─> 查询 inst_geo
                 ├─> 生成网格
                 ├─> 保存文件
                 └─> 更新 inst_geo.meshed
            
            (如果有负实体)
            └─> booleans_meshes_in_db()
                 ├─> 查询 neg_relate
                 ├─> 布尔运算
                 ├─> 保存结果
                 └─> 更新记录
```
