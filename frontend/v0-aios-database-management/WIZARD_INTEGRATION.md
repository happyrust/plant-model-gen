# 向导页面与任务创建功能集成方案

## 概述

将现有的 `http://localhost:3000/wizard` 向导页面与新的任务创建功能进行整合，提供统一的、功能更强大的任务创建体验。

## 集成策略

### 1. 双模式设计

**快速模式（默认）**
- 保持原有向导的简洁性
- 适合快速创建常见任务
- 支持多任务类型批量创建
- 简化的参数配置

**高级模式**
- 提供完整的任务创建向导
- 支持详细的参数配置
- 任务预览和资源预估
- 跳转到专门的高级创建页面

### 2. 功能整合

#### 原有向导功能保留
- ✅ 站点选择
- ✅ 任务类型多选
- ✅ 解析范围配置（全部/数据库编号/参考号）
- ✅ 优先级设置
- ✅ 任务名称输入

#### 新增功能
- ✅ 集成新的API接口
- ✅ 实时任务创建
- ✅ 错误处理和用户反馈
- ✅ 任务创建成功跳转
- ✅ 与任务监控系统集成

### 3. 用户体验优化

#### 界面改进
- 使用Tabs组件分离快速模式和高级模式
- 保持原有的步骤式布局
- 增强视觉反馈和状态指示
- 统一的错误处理机制

#### 交互优化
- 实时表单验证
- 智能默认值设置
- 批量任务创建支持
- 成功后的自动跳转

## 技术实现

### 1. 组件结构

```
app/wizard/page.tsx (增强版向导页面)
├── 快速模式 (TabsContent)
│   ├── 站点选择
│   ├── 任务类型配置
│   ├── 解析范围设置
│   └── 提交处理
└── 高级模式 (TabsContent)
    └── 跳转到 /task-creation
```

### 2. 状态管理

使用统一的 `useTaskCreation` Hook：
- 站点数据管理
- 任务创建API调用
- 错误处理
- 加载状态管理

### 3. API集成

```typescript
// 快速模式 - 批量创建任务
const taskPromises = selectedTaskTypes.map(async (taskType) => {
  const taskRequest = {
    taskName: formData.taskName || `${getTaskTypeName(taskType)}_${Date.now()}`,
    taskType: taskType,
    siteId: formData.siteId,
    priority: formData.priority,
    description: formData.description,
    parameters: {
      parseMode: parseMode,
      dbnum: config.dbnum,
      refno: config.refno,
      maxConcurrent: 1,
      parallelProcessing: false
    }
  }
  return await submitTask(taskRequest)
})
```

## 使用流程

### 快速模式流程

1. **访问向导页面** - `http://localhost:3000/wizard`
2. **选择站点** - 从可用站点列表中选择
3. **配置任务** - 选择任务类型和解析范围
4. **设置参数** - 优先级、任务名称等
5. **创建任务** - 批量创建多个任务
6. **监控任务** - 自动跳转到任务监控页面

### 高级模式流程

1. **切换到高级模式** - 点击"高级模式"标签
2. **跳转高级创建** - 点击"前往高级任务创建"
3. **使用完整向导** - 4步向导流程
4. **详细配置** - 完整的参数配置选项
5. **预览确认** - 资源预估和配置预览
6. **创建任务** - 创建单个精细化任务
