# 空间查询可视化功能文档

## 概述

空间查询可视化是一个强大的工具,用于查询和可视化AIOS系统中的空间、房间和构件的层级关系。

## 功能特性

### 1. 基础查询
- **参考号输入**: 输入参考号(Reference Number)来查询对应的节点
- **自动类型识别**: 系统自动识别输入的参考号对应的节点类型(空间/房间/构件)
- **层级展示**: 自动展示该节点的所有子节点

### 2. 三种可视化模式

#### 简单树形视图
- 最基础的树形展示
- 支持展开/折叠节点
- 显示节点的基本信息(名称、ID、类型)
- 适合快速浏览

#### 高级树形视图
- 包含搜索和过滤功能
- 支持按节点类型过滤(空间/房间/构件)
- 实时统计信息显示
- 支持展开/折叠全部节点
- 适合详细分析

#### 流程图视图
- 使用React Flow库实现
- 支持拖拽节点
- 支持缩放和平移
- 显示节点之间的连接关系
- 包含小地图导航
- 适合大规模数据展示

### 3. 交互功能

#### 节点展开/折叠
- 点击节点旁的箭头展开/折叠
- 支持动态加载子节点
- 显示加载状态

#### 搜索和过滤
- 按名称搜索
- 按类型搜索
- 按ID搜索
- 实时过滤结果

#### 统计信息
- 显示空间、房间、构件的数量
- 显示过滤后的节点数量
- 实时更新统计数据

## 使用指南

### 基本使用流程

1. **打开页面**
   - 导航到 `/spatial-visualization`
   - 或从侧边栏点击"空间查询" > "可视化查询"

2. **输入参考号**
   - 在输入框中输入参考号(例如: 24381)
   - 按Enter键或点击"查询"按钮

3. **查看结果**
   - 系统会显示该节点的信息
   - 自动展示其所有直接子节点

4. **选择可视化模式**
   - 点击"简单树形"、"高级树形"或"流程图视图"按钮
   - 切换不同的展示方式

### 高级树形视图使用

1. **搜索节点**
   - 在搜索框中输入关键词
   - 支持按名称、类型或ID搜索

2. **过滤节点**
   - 点击"全部"、"空间"、"房间"或"构件"按钮
   - 只显示选定类型的节点

3. **展开/折叠**
   - 点击节点旁的箭头展开/折叠
   - 点击"展开全部"按钮展开所有节点
   - 点击"折叠全部"按钮折叠所有节点

### 流程图视图使用

1. **拖拽节点**
   - 点击并拖动节点来调整位置

2. **缩放和平移**
   - 使用鼠标滚轮缩放
   - 按住空格键并拖动来平移

3. **小地图导航**
   - 右下角显示小地图
   - 点击小地图快速导航

## API集成

### 后端API端点

#### 1. 查询空间节点
```
GET /api/spatial/query/:refno
```

**响应示例:**
```json
{
  "success": true,
  "node": {
    "refno": 24381,
    "name": "Main Space",
    "noun": "FRMW",
    "node_type": "SPACE",
    "children_count": 5
  },
  "children": [
    {
      "refno": 24382,
      "name": "Room 1",
      "noun": "PANE",
      "node_type": "ROOM",
      "children_count": 10
    }
  ]
}
```

#### 2. 查询子节点
```
GET /api/spatial/children/:refno
```

**响应示例:**
```json
[
  {
    "refno": 24383,
    "name": "Component 1",
    "noun": "PIPE",
    "node_type": "COMPONENT",
    "children_count": 0
  }
]
```

#### 3. 获取节点详细信息
```
GET /api/spatial/node-info/:refno
```

## 数据模型

### 节点类型

| 类型 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| SPACE | 🏢 | 蓝色 | 空间/框架 |
| ROOM | 🚪 | 绿色 | 房间/面板 |
| COMPONENT | ⚙️ | 紫色 | 构件/元素 |

### 节点属性

- **refno**: 参考号(唯一标识)
- **name**: 节点名称
- **noun**: 节点类型代码(FRMW, PANE, PIPE等)
- **node_type**: 分类类型(SPACE, ROOM, COMPONENT)
- **children_count**: 子节点数量

## 性能优化

### 动态加载
- 子节点按需加载,不会一次性加载所有数据
- 减少初始加载时间

### 缓存机制
- 已加载的子节点会被缓存
- 避免重复查询

### 虚拟化渲染
- React Flow使用虚拟化渲染
- 支持大规模数据展示

## 故障排除

### 查询失败
- 检查参考号是否正确
- 确保后端API服务正常运行
- 查看浏览器控制台的错误信息

### 节点不显示
- 检查节点是否有子节点
- 尝试切换可视化模式
- 刷新页面重新加载

### 性能问题
- 减少搜索范围
- 使用过滤功能
- 切换到简单树形视图

## 技术栈

- **前端框架**: Next.js 14 + React 18
- **UI组件**: Radix UI + Tailwind CSS
- **可视化库**: React Flow
- **状态管理**: React Hooks
- **API通信**: Fetch API

## 开发指南

## 常见问题

**Q: 如何查询一个空间的所有房间?**
A: 输入空间的参考号,系统会自动显示该空间的所有房间。

**Q: 如何查询一个房间的所有构件?**
A: 输入房间的参考号,系统会自动显示该房间的所有构件。

**Q: 支持批量查询吗?**
A: 目前不支持,但可以逐个查询。

**Q: 可以导出查询结果吗?**
A: 目前不支持,但可以截图或复制数据。
