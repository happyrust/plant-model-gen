# 解析模型配置界面管理功能开发 - 技术架构说明

## 一、技术架构概述

### 1.1 前端技术栈

**核心框架：**
- **Next.js 14** - React 全栈框架，提供 SSR/SSG 和 API Routes
- **React 18** - 用户界面库，使用函数组件和 Hooks
- **TypeScript** - 类型安全的 JavaScript 超集

**UI 组件库：**
- **Radix UI** - 无样式、可访问的组件原语库
  - Dialog、Select、RadioGroup、Progress、Tabs 等
- **Tailwind CSS** - 实用优先的 CSS 框架
- **Lucide React** - 图标库

**表单与验证：**
- **React Hook Form** - 高性能表单库
- **Zod** - TypeScript 优先的 schema 验证库
- **@hookform/resolvers** - React Hook Form 与 Zod 集成

**状态管理：**
- **React Hooks** - useState、useCallback、useEffect
- **自定义 Hooks** - 封装业务逻辑和 API 调用

**实时通信：**
- **WebSocket** - 实时双向通信（通过自定义 `useWebSocket` Hook）

**数据可视化：**
- **Recharts** - React 图表库（用于系统监控）
- **ReactFlow** - 流程图可视化（用于空间关系展示）

### 1.2 后端技术栈

**核心服务：**
- **Rust** - 系统级编程语言
- **Axum** - 异步 Web 框架
- **SurrealDB** - 图数据库（主要数据存储）
- **SQLite** - 轻量级关系数据库（任务元数据）

**API 架构：**
- **RESTful API** - HTTP/JSON 接口
- **WebSocket** - 实时状态推送
- **gRPC** - 内部服务通信（可选）

## 二、功能模块详细说明

### 2.1 模型解析配置表单

#### 2.1.1 组件架构

**主组件：`TaskCreationWizard`**
- 位置：`components/task-creation/TaskCreationWizard.tsx`
- 职责：多步骤向导式表单，引导用户完成任务配置

**步骤划分：**
1. **步骤 1：基础信息配置**
   - 任务名称（实时验证唯一性）
   - 任务类型选择（5 种类型：数据解析、模型生成、空间树生成、全量同步、增量同步）
   - 优先级设置（低/普通/高/紧急）
   - 任务描述（可选）

2. **步骤 2：选择部署站点**
   - 从可用站点列表中选择
   - 显示站点状态和环境信息
   - 支持站点详情查看

3. **步骤 3：任务参数配置**
   - 根据任务类型动态显示参数表单
   - **数据解析任务参数：**
     - 解析模式：全部解析 / 指定数据库编号 / 指定参考号
     - 数据库编号（条件显示）
     - 参考号（条件显示）
   - **模型生成任务参数：**
     - 生成选项：3D模型 / 网格 / 空间树 / 布尔运算（多选）
     - 网格容差比例（数值输入）
   - **通用参数：**
     - 最大并发数（1-10）
     - 并行处理开关

4. **步骤 4：预览和确认**
   - 任务配置预览
   - 资源需求预估（内存/CPU/磁盘）
   - 注意事项和警告
   - 最终确认创建

#### 2.1.2 表单验证机制

**实时验证：**
```typescript
// 任务名称验证
const handleNameValidation = async (name: string) => {
  const result = await validateName(name)
  setNameValidation(result) // 显示验证结果
}
```

**步骤验证：**
```typescript
const validateCurrentStep = (): boolean => {
  switch (currentStep) {
    case 1: return formData.taskName.trim().length > 0 && 
                 Boolean(formData.taskType) && 
                 (nameValidation?.available ?? false)
    case 2: return formData.siteId.trim().length > 0
    case 3: return true // 参数配置总是有效
    case 4: return true // 预览步骤总是有效
  }
}
```

**API 验证：**
- `GET /api/task-creation/validate-name?name={name}` - 验证任务名称唯一性
- `POST /api/task-creation/preview` - 预览配置并获取资源预估

#### 2.1.3 数据流

```
用户输入 → 表单状态更新 → 实时验证 → UI 反馈
    ↓
步骤完成 → 验证通过 → 允许进入下一步
    ↓
所有步骤完成 → 预览配置 → 提交创建 → API 调用 → 跳转监控页面
```

### 4.2 实时通信

**WebSocket Hook：**
```typescript
export function useWebSocket(url: string) {
  const [isConnected, setIsConnected] = useState(false)
  const [lastMessage, setLastMessage] = useState(null)
  
  useEffect(() => {
    const ws = new WebSocket(url)
    ws.onmessage = (event) => setLastMessage(JSON.parse(event.data))
    ws.onopen = () => setIsConnected(true)
    ws.onclose = () => setIsConnected(false)
    return () => ws.close()
  }, [url])
  
  return { isConnected, lastMessage }
}
```
