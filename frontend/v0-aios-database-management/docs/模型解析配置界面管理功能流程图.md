## 五、数据流详细流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant UI as UI组件
    participant H as Hook
    participant API as API客户端
    participant WS as WebSocket
    participant B as 后端API
    participant DB as 数据库
    participant E as 执行引擎
    
    Note over U,E: 任务创建流程
    U->>UI: 填写表单
    UI->>H: 更新表单状态
    H->>API: 验证任务名称
    API->>B: GET /api/task-creation/validate-name
    B->>DB: 查询任务名称
    DB-->>B: 返回结果
    B-->>API: 验证结果
    API-->>H: 验证结果
    H-->>UI: 更新UI状态
    
    U->>UI: 提交表单
    UI->>H: submitTask()
    H->>API: POST /api/task-creation
    API->>B: 创建任务请求
    B->>DB: 保存任务配置
    B->>E: 启动任务执行
    DB-->>B: 保存成功
    B-->>API: 返回任务ID
    API-->>H: 创建成功
    H-->>UI: 显示成功提示
    UI->>U: 跳转监控页面
    
    Note over U,E: 实时监控流程
    UI->>WS: 建立WebSocket连接
    WS->>B: 连接请求
    B-->>WS: 连接成功
    
    E->>B: 任务状态更新
    B->>WS: 推送更新消息
    WS->>H: 接收消息
    H->>UI: 更新任务状态
    UI->>U: 刷新显示
    
    Note over U,E: 任务操作流程
    U->>UI: 点击启动/停止/暂停
    UI->>H: 调用操作函数
    H->>API: POST /api/tasks/{id}/action
    API->>B: 执行操作
    B->>E: 控制任务执行
    E-->>B: 操作结果
    B-->>API: 返回结果
    API-->>H: 操作成功
    H->>UI: 更新状态
    UI->>U: 显示更新
```

## 九、WebSocket实时通信流程

```mermaid
sequenceDiagram
    participant UI as 前端UI
    participant Hook as useWebSocket Hook
    participant WS as WebSocket连接
    participant Server as 后端服务器
    participant Engine as 任务执行引擎
    
    UI->>Hook: 初始化Hook
    Hook->>WS: 创建WebSocket连接
    WS->>Server: ws://host/ws/tasks/updates
    
    Server-->>WS: 连接成功
    WS-->>Hook: onopen事件
    Hook-->>UI: 更新连接状态为已连接
    
    loop 任务执行中
        Engine->>Server: 任务状态更新
        Server->>WS: 推送消息
        Note over Server,WS: {type: 'task_update', data: {...}}
        WS->>Hook: onmessage事件
        Hook->>Hook: 解析消息
        Hook->>UI: 更新任务状态
        UI->>UI: 刷新显示
    end
    
    loop 系统指标更新
        Server->>WS: 推送系统指标
        Note over Server,WS: {type: 'system_metrics', data: {...}}
        WS->>Hook: onmessage事件
        Hook->>UI: 更新系统指标
        UI->>UI: 刷新图表
    end
    
    alt 连接断开
        WS->>Hook: onclose事件
        Hook->>UI: 更新连接状态为断开
        Hook->>Hook: 尝试重连
        Hook->>WS: 重新建立连接
    end
    
    UI->>Hook: 组件卸载
    Hook->>WS: 关闭连接
    WS->>Server: 断开连接
```

## 流程图说明

### 图例说明

- **黄色节点**：决策点或条件判断
- **绿色节点**：成功状态或正向流程
- **红色节点**：错误状态或异常流程
- **蓝色节点**：用户操作或输入
- **橙色节点**：系统处理或业务逻辑
- **紫色节点**：数据存储

### 使用说明

1. **任务创建流程**：展示了从用户访问页面到任务创建完成的完整流程
2. **站点创建流程**：展示了三步向导式站点创建的详细步骤
3. **任务监控流程**：展示了实时监控和轮询更新的机制
4. **系统架构流程**：展示了前后端分层架构和数据流向
5. **数据流流程**：使用序列图展示API调用和WebSocket通信的时序
6. **错误处理流程**：展示了各种错误情况的处理机制
7. **站点管理流程**：展示了站点管理的各种操作流程
8. **任务状态流转**：展示了任务状态的状态机转换
9. **WebSocket流程**：展示了实时通信的建立和维护
10. **表单验证流程**：展示了前端表单验证的完整流程

### 技术要点

- **实时性**：通过WebSocket实现任务状态的实时更新
- **可靠性**：通过轮询机制作为WebSocket的备选方案
- **用户体验**：通过实时验证和进度反馈提升用户体验
- **错误处理**：完善的错误处理和用户提示机制
- **状态管理**：清晰的状态流转和状态管理
