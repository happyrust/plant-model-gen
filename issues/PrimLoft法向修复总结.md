## PrimLoft Sweep 法向修复记要

在导出 GENSEC 25688_76336 时，PrimLoft 扫掠体的端盖与侧壁法向被错误地写成接近 `+Z`，渲染效果如同上图：灰色翼板外侧光照正常，但内侧泛白，像被强光直射的错位空槽。为消除这条“飘白光带”，本次修复专注于重建扫掠几何的法向生成流程。

### 问题诊断

1. **端盖法向偏转**：`drns/drne` 原被直接当作端盖法向使用，但它们与扫掠方向近乎垂直，导致盖板向 `±Z` 指向天空。OBJ 里所有 cap 顶点都写入 `(0, 0, ±1)`。
2. **侧壁法向共用顶点**：扫掠轮廓的相邻边共用同一顶点法向，任何折角都会被“抹平”，渲染器分不出每段面的真实朝向。

### 实施方案

| 步骤 | 处理要点 |
| --- | --- |
| 端面投影 | 引入 `compute_path_offset`，将局部坐标点沿扫掠方向投影到 `drns/drne` 定义的斜切平面上，重新布局端盖顶点，使平面与路径对齐。 |
| 方向校验 | 新增 `polygon_signed_area` 判断截面顺逆时针；依据扫掠方向自动翻转 cap 法向，避免漏乘 `-1`。 |
| 侧壁拆分 | 不再复用截面环顶点，而是按轮廓边逐段构建四边形并为每段写入独立法向，确保持面呈现硬边角度。 |
| 容错保护 | 为零长度路径、零向量法向增加早退与归一化校验，防止异常参数生成 NaN。 |

### 结果验证

```bash
cargo run --example test_export_gensec_obj
```

导出的 OBJ 中，端盖法向与扫掠方向同轴（`±X`），侧壁法向按照轮廓外法正确指向。渲染效果已从“光带错位”恢复为图中灰色槽钢般的层次分明：外侧受光、内侧阴影，几何不再透亮发白。

---

> 视觉比喻：原模型像一条被阳光打穿的折纸；修复后它回到了扎实的槽钢，灰面吸光、白底反光，法向和光照重新贴合结构。
