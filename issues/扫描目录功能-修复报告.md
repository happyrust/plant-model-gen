# 扫描目录功能修复报告

## 📋 问题描述

**问题**: 在创建部署站点向导的第2步，扫描目录 `/Volumes/DPC/work/e3d_models` 时失败，显示"扫描目录失败"错误。

**错误信息**:
```
Error: 扫描目录失败
  at scanDirectory (webpack-internal:///(app-pages-browser)/.../hooks/use-site-operations.ts:23:15)
  at async handleScanDirectory (webpack-internal:///(app-pages-browser)/.../components/deployment-sites/enhanced-create-site-dialog.tsx:60:13)
```

**根本原因**: 
`hooks/use-site-operations.ts` 中的 `scanDirectory` 函数直接拼接URL，没有使用项目统一的 `buildApiUrl` 函数，导致当 `NEXT_PUBLIC_API_BASE_URL` 未设置时，URL格式不正确。

---

## 🔧 修复方案

### 修改的文件

#### 1. `lib/api.ts`
**修改内容**: 导出 `buildApiUrl` 函数

```typescript
/**
 * 构建完整的API URL
 * @param path API路径，必须以 / 开头
 * @returns 完整的API URL
 */
export function buildApiUrl(path: string): string {
  if (!path.startsWith("/")) {
    throw new Error(`API 路径必须以 / 开头: ${path}`)
  }
  if (!API_BASE_URL) {
    return path
  }
  return `${API_BASE_URL}${path}`
}
```

**变更**: 
- ✅ 添加 `export` 关键字
- ✅ 添加 JSDoc 注释
- ✅ 添加返回类型注解

---

#### 2. `hooks/use-site-operations.ts`
**修改内容**: 使用 `buildApiUrl` 函数

**修改前**:
```typescript
import { createDeploymentSite, type CreateDeploymentSitePayload } from "@/lib/api"

export async function scanDirectory(
  rootDirectory: string,
  setProjects: (projects: ProjectInfo[]) => void
): Promise<void> {
  const params = new URLSearchParams({
    directory_path: rootDirectory,
    recursive: "true",
    max_depth: "4",
  })

  const baseUrl = process.env.NEXT_PUBLIC_API_BASE_URL?.replace(/\/$/, "") || ""
  const requestUrl = `${baseUrl}/api/wizard/scan-directory?${params.toString()}`

  const response = await fetch(requestUrl)
  // ...
}
```

**修改后**:
```typescript
import { createDeploymentSite, type CreateDeploymentSitePayload, buildApiUrl } from "@/lib/api"

export async function scanDirectory(
  rootDirectory: string,
  setProjects: (projects: ProjectInfo[]) => void
): Promise<void> {
  const params = new URLSearchParams({
    directory_path: rootDirectory,
    recursive: "true",
    max_depth: "4",
  })

  const requestUrl = buildApiUrl(`/api/wizard/scan-directory?${params.toString()}`)

  const response = await fetch(requestUrl)
  // ...
}
```

**变更**:
- ✅ 导入 `buildApiUrl` 函数
- ✅ 删除手动URL拼接逻辑
- ✅ 使用统一的 `buildApiUrl` 函数

## 🔍 技术细节

### URL构建逻辑

**修复前的问题**:
```typescript
const baseUrl = process.env.NEXT_PUBLIC_API_BASE_URL?.replace(/\/$/, "") || ""
const requestUrl = `${baseUrl}/api/wizard/scan-directory?${params.toString()}`
```

当 `NEXT_PUBLIC_API_BASE_URL` 未设置时:
- `baseUrl` = `""`
- `requestUrl` = `"/api/wizard/scan-directory?..."`
- 但是手动拼接可能导致双斜杠或其他格式问题

**修复后**:
```typescript
const requestUrl = buildApiUrl(`/api/wizard/scan-directory?${params.toString()}`)
```

`buildApiUrl` 函数统一处理:
- 验证路径以 `/` 开头
- 如果 `API_BASE_URL` 为空，直接返回路径
- 如果 `API_BASE_URL` 有值，拼接完整URL
- 保证URL格式一致性
