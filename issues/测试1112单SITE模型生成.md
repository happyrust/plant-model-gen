# 测试1112数据库单个SITE模型生成

## 📋 任务目标

测试生成1112数据库中单个SITE的3D模型,验证模型生成流程的正确性。

---

## 🎯 执行方案

### 方案概述
创建专用测试示例,精确控制只生成一个SITE的模型,避免生成所有SITE导致的长时间等待。

### 核心步骤
1. 查询1112数据库的SITE列表
2. 选择一个SITE进行测试
3. 使用 `generate_xtk_by_dbno_refnos` 一步完成:
   - 生成几何数据并保存到SurrealDB
   - 从SurrealDB查询数据
   - 生成XKT文件
4. 验证生成结果

### 技术路线
使用现有的 `generate_xtk_by_dbno_refnos` 函数,流程简单清晰:
```rust
// 一步完成所有操作
generate_xtk_by_dbno_refnos(
    1112,                    // 数据库编号
    vec![site_refno],        // SITE的refno
    "output/site.xkt",       // 输出路径
    true,                    // 压缩
    &db_option,              // 配置
).await?;
```

## 🔧 故障排查

### 问题1: 未找到SITE节点

**原因**: 数据未同步到SurrealDB

**解决**:
```bash
# 1. 确保 DbOption.toml 配置正确
grep -E "total_sync|manual_db_nums|included_db_files" DbOption.toml

# 2. 运行数据同步
cargo run

# 3. 重新查询
cargo run --example query_1112_sites
```

---

### 问题2: 编译错误

**错误**: `cannot find function 'get_descendants_by_types'`

**原因**: 缺少必要的feature

**解决**:
```bash
# 使用完整的feature编译
cargo run --example test_1112_single_site --features gen_model
```

---

### 问题3: 生成的refno数量为0

**原因**: SITE下没有几何体,或查询失败

**调试**:
1. 检查SITE是否有子节点
2. 查看控制台错误信息
3. 尝试使用其他SITE的refno

---

### 问题4: XKT文件大小为0

**原因**: 几何数据生成失败

**解决**:
1. 检查 `gen_model = true` 和 `gen_mesh = true`
2. 查看生成过程的错误信息
3. 检查网格文件目录权限: `assets/meshes`

## 🎓 技术说明

### 查询流程
```rust
// 1. 查询SITE
let sites = query_by_type(&["SITE"], 1112, None).await?;

// 2. 获取子孙节点(可选,用于统计)
let descendants = get_descendants_by_types(
    site_refno,
    &[], // 所有类型
    Some(20) // 最大深度
).await?;

// 3. 一步生成模型和XKT
generate_xtk_by_dbno_refnos(
    1112,
    vec![site_refno],
    "output/site.xkt",
    true,
    &db_option,
).await?;
```

### 数据流
```
PDMS文件 → SurrealDB (PE数据)
                ↓
         查询SITE refno
                ↓
    generate_xtk_by_dbno_refnos
                ↓
    ┌───────────┴───────────┐
    ↓                       ↓
生成几何数据          从SurrealDB查询
保存到SurrealDB       inst_relate/inst_geo
(inst_relate,              ↓
 inst_geo,            转换为XKT格式
 meshes)                   ↓
                      输出XKT文件
```
