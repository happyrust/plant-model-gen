# 自动启动解析任务（方案A）

## 背景
- Web-UI 部署站点详情页中，“启动解析任务”按钮仅创建任务（状态 Pending），随后跳转到 /tasks。
- 后端真正启动执行依赖 POST /api/tasks/:id/start 将任务置为 Running，并 tokio::spawn 后台执行。

## 目标
- 前端在创建解析任务成功后，立即调用启动接口，使任务进入 Running，再跳转任务页。
- 不改变后端 create_task 的语义，影响面最小。

## 变更点
- 文件：src/web_ui/static/projects.js
- 函数：window.launchParsingTask
- 修改：在收到 createResp.ok 后，解析返回 JSON（含任务 id），随后调用
  POST /api/tasks/:id/start 启动任务；若启动失败，提示错误并中止跳转。

## 验证步骤
1. cargo run 启动 web-ui（调试模式）
2. 打开 /deployment-sites，选中某站点点击“启动解析任务”
3. 跳转 /tasks 后应看到任务状态为 Running，并持续刷新进度/日志

## 备注
- 本次仅对“解析任务”进行自动启动。如需“建模任务/空间索引任务”也采用相同策略，可复用同样改造方式。
