> - `src/web_server/mod.rs` - 启动时调用初始化检查

## 需求概述

实现 Scene Tree 的初始化方法，在 Web Server 启动时检查是否已初始化，如果未初始化则自动执行初始化。初始化时同步所有节点，对于几何节点需要区分正实体节点和负实体节点。

|------|------|------|

| parent | option<int> | 父节点 ID |

| dbno | int | 数据库编号 |

### 缺失功能

1. **geo_type 字段**：无法区分正实体和负实体
2. **初始化检查**：无启动时自动检查机制
3. **Web Server 集成**：启动流程未包含 scene_tree 初始化

## 设计方案

### 1. Schema 扩展 - 新增 geo_type 字段

**文件**: `src/scene_tree/schema.rs`

在 `scene_node` 表中新增 `geo_type` 字段：

```sql
-- 几何类型（用于区分正负实体）
-- 可选值: Pos, Neg, CataNeg, CataCrossNeg, Compound, CatePos, DesiPos, None
DEFINE FIELD IF NOT EXISTS geo_type ON TABLE scene_node TYPE option<string>;

-- 新增索引
DEFINE INDEX IF NOT EXISTS idx_geo_type ON TABLE scene_node COLUMNS geo_type;
```

**geo_type 取值说明**：

| 类型 | 含义 | 用途 |
|------|------|------|
| `Pos` | 正实体（可见） | 普通的正几何体 |
| `Neg` | 负实体 | 设计级别的负几何体（PRIM/LOOP/PLOO） |
| `Compound` | 复合实体 | 有关联负实体的正实体（需要合并） |
| `CataNeg` | 元件库负实体 | 来自 CATE 元件库的负几何体 |
| `CataCrossNeg` | 交叉负实体 | 来自 NGMR 的跨元素负实体 |
| `CatePos` | 被布尔替换的原始正实体 | 布尔运算后隐藏的原始正实体 |
| `DesiPos` | 设计型正实体 | 设计级别的正实体 |

async fn get_geo_type_by_refno(refno: RefnoEnum) -> Result<Option<String>> {
    // 查询 geo_relate 表获取 geo_type
    let sql = format!(

### 3. 启动时初始化检查

**文件**: `src/scene_tree/mod.rs`

新增函数：

```rust
/// 检查 scene_tree 是否已初始化
pub async fn is_initialized() -> Result<bool> {
    let sql = "SELECT count() FROM scene_node GROUP ALL";
    let result: Vec<i64> = SUL_DB.query_take(&sql, 0).await?;
    Ok(result.first().copied().unwrap_or(0) > 0)
}

/// 确保 scene_tree 已初始化（启动时调用）
pub async fn ensure_initialized() -> Result<()> {
    // 1. 初始化 Schema（幂等）
    schema::init_schema().await?;

    // 2. 检查是否已有数据
    if is_initialized().await? {
        println!("[scene_tree] 已初始化，跳过");
        return Ok(());
    }

    // 3. 执行初始化
    println!("[scene_tree] 未初始化，开始全量同步...");
    init_scene_tree("ALL", false).await?;
    Ok(())
}
```

### 4. Web Server 启动集成

**文件**: `src/web_server/mod.rs`

在 `start_web_server_with_config` 函数中，数据库连接成功后添加：

```rust
// 在 ensure_projects_schema() 之后添加
// 确保 Scene Tree 已初始化
println!("🌳 检查 Scene Tree 初始化状态...");
match crate::scene_tree::ensure_initialized().await {
    Ok(_) => println!("✅ Scene Tree 初始化检查完成"),
    Err(e) => {
        eprintln!("⚠️ Scene Tree 初始化失败: {}", e);
        // 不阻塞启动，允许后续手动初始化
    }
}
```

## 注意事项

1. **性能考虑**：全量同步可能耗时较长，启动时不阻塞主流程，失败时仅打印警告
2. **幂等性**：Schema 使用 `IF NOT EXISTS`，重复执行不会报错
3. **geo_type 来源**：从 `geo_relate` 表查询，如果节点没有几何数据则为 `None`
4. **向后兼容**：`geo_type` 字段为可选，不影响现有查询逻辑
