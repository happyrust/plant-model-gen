# Prepack LOD 导出流程说明

## 概述

Prepack LOD 导出功能将 SurrealDB 中的模型数据导出为前端可用的 GLB + JSON 格式，支持多级 LOD 和实例化渲染。

---

## 命令行使用

```bash
# 导出全部模型
cargo run --bin aios-database -- \
  --config DbOption \
  --export-all-relates \
  --output ./output/ams-model

# 导出指定 refnos（调试模式）
cargo run --bin aios-database -- \
  --config DbOption \
  --export-all-relates \
  --export-refnos 24381_46955,24381_46960 \
  --output ./output/debug-model
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `--config` | 数据库配置文件名（不含扩展名） |
| `--export-all-relates` | 启用导出模式 |
| `--export-refnos` | 可选，指定要导出的 refno（逗号分隔） |
| `--output` | 输出目录路径 |
| `--export-all-lods` | 可选，导出所有 LOD 级别 |

## 核心数据结构

### GeomInstQuery (数据库查询结果)

```rust
pub struct GeomInstQuery {
    pub refno: RefnoEnum,           // 构件编号
    pub owner: RefnoEnum,           // 所属构件编号
    pub world_trans: PlantTransform, // 世界变换
    pub insts: Vec<ModelHashInst>,  // 几何实例列表
    pub has_neg: bool,              // 是否包含负实体
    pub spec_value: Option<i64>,    // 规格值
}
```

### ComponentRecord (收集后的构件记录)

```rust
pub struct ComponentRecord {
    pub refno: RefnoEnum,
    pub noun: String,
    pub name: Option<String>,
    pub world_transform: DMat4,      // refno 世界变换
    pub geometries: Vec<GeometryInstance>,
    pub owner_refno: Option<RefnoEnum>,
    pub owner_noun: Option<String>,
    pub spec_value: Option<i64>,
}
```

### GeometryInstance (几何体实例)

```rust
pub struct GeometryInstance {
    pub geo_hash: String,
    pub local_transform: DMat4,  // 几何体局部变换
    pub index: usize,
    pub unit_flag: bool,
}
```

## 变换矩阵处理

### 双层变换架构

```
数据库存储:
  inst_relate.world_trans  →  refno 的世界变换
  geo_relate.trans         →  几何体的局部变换

导出到 JSON:
  refno_transform = world_trans
  geo_transform   = geo_trans

前端渲染:
  最终矩阵 = refno_transform × geo_transform
```

### 为什么分离变换？

1. **复用几何体**：相同 geo_hash 可在不同位置复用
2. **层级控制**：可单独修改 refno 位置而不影响几何体
3. **渲染优化**：支持 GPU 实例化渲染

```

├── lod_L1.glb              # LOD 1 几何体合集
```
