# build_instances_payload 快速参考

## 🎯 一句话总结

`build_instances_payload` 函数将工厂 3D 模型数据按 BRAN/EQUI 分组，为每个构件生成实例数据（包含矩阵、颜色、LOD等），最终输出 JSON 格式的实例清单。

---

## 📊 流程速览

```
初始化 → 收集所有者 → 分组处理 → 构建层级 → 生成实例 → JSON序列化
```

---

## 🔑 关键概念

| 概念 | 说明 |
|------|------|
| **BRAN** | 管道分支，可包含多个构件和TUBI |
| **EQUI** | 设备，可包含多个构件 |
| **TUBI** | 管道，属于BRAN，有顺序关系 |
| **NameTable** | 名称去重表，避免重复存储 |
| **ColorPalette** | 颜色调色板，按构件类型分配 |
| **LOD Mask** | 32位掩码，表示该实例在哪些LOD级别可用 |
| **geo_hash** | 几何体唯一标识 |
| **geo_index** | 几何体在索引中的位置 |

## 📈 性能优化

| 优化 | 效果 |
|------|------|
| **名称去重** | 减少 90% 的名称存储 |
| **颜色去重** | 减少 75% 的颜色存储 |
| **预计算索引** | O(1) 查询几何体索引 |
| **使用引用** | 避免数据复制 |
| **BTreeMap** | 保持TUBI顺序 |

---

## 🐛 常见问题

### Q1: 为什么使用 BTreeMap 而不是 HashMap？
**A**: TUBI 需要保持顺序（表示管道连接关系），BTreeMap 自动排序。

### Q2: 什么是 LOD 掩码？
**A**: 32位整数，每一位表示一个LOD级别。bit 0 = L1, bit 1 = L2, 等等。

### Q3: 矩阵为什么要缩放？
**A**: 共享几何体需要在矩阵中应用单位转换，专用几何体已在顶点中转换。

### Q4: 名称表的作用是什么？
**A**: 避免重复存储相同名称，只存储一次，通过索引引用。

### Q5: 如何判断是共享还是专用几何体？
**A**: 检查 geo_hash 是否包含下划线。纯数字 = 共享，包含下划线 = 专用。
