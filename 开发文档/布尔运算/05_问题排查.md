# 布尔运算问题排查

## 1. 常见问题

### 问题 1: 负实体没有被减去

**症状**: 模型没有孔洞

**排查步骤**:

```bash
# 1. 检查负实体关系是否存在
cargo run --bin test_boolean_debug --features gen_model
```

```sql
-- 2. 查询 neg_relate 数量
SELECT count() FROM neg_relate;

-- 3. 查询特定实例的负实体
SELECT * FROM neg_relate WHERE out = pe:{refno};
```

**常见原因**:

- `neg_relate_map` 为空（负实体未收集）
- 配置 `full_noun_enabled_categories` 未包含负实体类别
- 数据源没有负实体数据

### 问题 2: bad_bool = true

**排查 SQL**:

```sql
-- 查询失败的布尔运算
SELECT in, bad_bool FROM inst_relate WHERE bad_bool = true LIMIT 10;
```

**常见原因**:

- 正实体 mesh 加载失败
- 负实体 mesh 加载失败
- 几何体无效（三角形数量为 0）

### 问题 3: booled_id 未更新

**排查 SQL**:

```sql
-- 查询有负实体但未完成布尔的实例
SELECT in FROM inst_relate 
WHERE ((in<-neg_relate)[0] != NONE OR (in<-ngmr_relate)[0] != NONE)
  AND booled_id = NONE
  AND bad_bool != true;
```

### 问题 4: 正实体 manifold 没有三角形

**症状**: 错误日志 `布尔运算失败: 正实体 manifold 没有三角形`

**根因分析**:

PlantMesh 使用**非共享顶点模式**（每个三角形有独立的顶点用于硬表面渲染），但 Manifold 库要求**共享顶点拓扑**才能识别为有效的 2-流形。

**解决方案**:

在 `rs-core/src/geometry/csg.rs` 的 `weld_vertices_for_manifold` 函数中实现了**容差顶点焊接**：

```rust
// 使用量化整数键（而非 to_bits() 精确比较）
let quantize = |v: Vec3| -> (i64, i64, i64) {
    (
        (v.x * precision).round() as i64,
        (v.y * precision).round() as i64,
        (v.z * precision).round() as i64,
    )
};
```

**关键改进**:
1. **自适应精度**: 根据 mesh AABB 自动选择量化精度
2. **容差比较**: 数值接近的顶点正确合并
3. **退化三角形过滤**: 移除顶点重合的无效三角形

### 问题 5: 负实体坐标偏移很大（carrier_world_trans 问题）

**症状**: 布尔运算无效果，负实体位置完全不在正实体范围内

**调试输出**:
```
pos_world_trans: DVec4(-3360.0, -21150.0, 3290.0, 1.0)
carrier_world: DVec4(0.0, 0.0, 0.0, 1.0)  ← 全是0！
relative: DVec4(21150.0, ...)  ← 偏移巨大
```

**根因**:
在 `boolean_query_optimized.rs` 中，所有 `neg_infos` 被放入一个虚拟 carrier，使用 `PlantTransform::default()`（单位矩阵），导致实际的 `carrier_world_trans` 被忽略。

**解决方案**:
在 `manifold_bool.rs` 中直接使用 `NegInfo.carrier_world_trans`：

```rust
let carrier_world_mat = carrier_world_trans
    .as_ref()
    .map(|t| t.0.to_matrix().as_dmat4())
    .unwrap_or(DMat4::IDENTITY);
```

### 问题 6: 导出 OBJ 模型位置错误（变换重复应用）

**症状**: 导出的 OBJ 模型位置乱七八糟

**根因**:
对于 `booled_id` 的查询返回 `transform = world_trans.d`，但导出代码又做了 `world_transform × local_transform`，导致世界变换被应用两次。

**解决方案**:
在 `ComponentRecord` 添加 `has_neg` 字段，导出时根据此字段决定变换方式：

```rust
let combined_transform = if component.has_neg {
    instance.local_transform  // booled_id: 已包含 world_trans
} else {
    component.world_transform * instance.local_transform  // 普通几何体
};
```

  count() as total,
  count(booled = true) as cata_done,
  count(booled_id != NONE) as inst_done,
  count(bad_bool = true) as failed
FROM inst_relate;

-- 负实体关系统计
SELECT count() as neg_count FROM neg_relate;
