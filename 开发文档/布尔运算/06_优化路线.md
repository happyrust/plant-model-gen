# 布尔运算性能优化路线

> 更新时间: 2024-12-19

## 1. 当前架构瓶颈

### 1.1 瓶颈识别

| 层级 | 瓶颈点 | 影响 | 代码位置 |
|------|--------|------|----------|
| **I/O 层** | 逐个 mesh 文件加载 `load_manifold()` | 频繁磁盘 I/O | `manifold_bool.rs:107` |
| **计算层** | 顺序执行布尔运算 `for query in queries` | CPU 未充分利用 | `manifold_bool.rs:456` |
| **数据库层** | 批量查询后逐个处理 | 查询合并但执行串行 | `manifold_bool.rs:373` |
| **内存层** | 重复加载相同 mesh（负实体可被多个正实体使用） | 内存带宽浪费 | `manifold_bool.rs:107` |

### 1.2 当前执行流程

```
apply_insts_boolean_manifold
  └─ for query in queries  ← 顺序遍历（瓶颈）
       └─ apply_boolean_for_query
            ├─ load_manifold (正实体)  ← 磁盘 I/O
            ├─ load_manifold (负实体)  ← 磁盘 I/O（可能重复）
            ├─ batch_boolean_subtract   ← CPU 密集
            └─ update_booled_result     ← DB 写入（逐条）
```

### 1.3 已有优化

- ✅ BRAN 过滤已按任务类型细化（MemoryTasks 路径）
  - `BRAN + InstNeg`: 跳过
  - `BRAN + CataNeg`: 按数据执行（不再一刀切跳过）
- ✅ AABB 哈希存储避免重复计算
- ✅ 批量查询负实体 `query_manifold_boolean_operations_batch_optimized`

## 2. 优化方案

### 方案 1：并发执行布尔运算 ⭐⭐⭐

**目标**: 利用多核 CPU 并行处理布尔运算

**改造位置**: `manifold_bool.rs` - `apply_insts_boolean_manifold`

**当前代码**:
```rust
for query in queries {
    apply_boolean_for_query(query, replace_exist).await?;
}
```

**优化后**:
```rust
use futures::stream::{self, StreamExt};

const CONCURRENCY: usize = 8; // 可通过 DbOption 配置

stream::iter(queries)
    .map(|query| apply_boolean_for_query(query, replace_exist))
    .buffer_unordered(CONCURRENCY)
    .collect::<Vec<_>>()
    .await;
```

**预期收益**: 8 核 CPU 可提升 4-6x

**风险**: 内存占用增加，需监控

---

### 方案 2：Mesh 缓存池 ⭐⭐⭐

**目标**: 避免同一 mesh 被多个正实体重复加载

**改造位置**: `manifold_bool.rs` - 新增缓存模块

**实现**:
```rust
use dashmap::DashMap;
use once_cell::sync::Lazy;

static MANIFOLD_CACHE: Lazy<DashMap<String, ManifoldRust>> = Lazy::new(DashMap::new);

fn load_manifold_cached(id: &str, mat: DMat4, more_precision: bool) -> anyhow::Result<ManifoldRust> {
    // 缓存 key 不含变换矩阵，变换在获取后应用
    if let Some(cached) = MANIFOLD_CACHE.get(id) {
        let cloned = cached.clone();
        return Ok(cloned.transform(mat));
    }
    
    let manifold = load_manifold_internal(id)?;
    MANIFOLD_CACHE.insert(id.to_string(), manifold.clone());
    Ok(manifold.transform(mat))
}

pub fn clear_manifold_cache() {
    MANIFOLD_CACHE.clear();
}
```

**预期收益**: 减少 50-80% 重复磁盘 I/O

**注意**: 需要在布尔运算批次结束后清理缓存，避免内存泄漏

---

### 方案 3：批量数据库写入 ⭐⭐

**目标**: 减少数据库往返次数

**改造位置**: `manifold_bool.rs` - `apply_boolean_for_query` 返回结果后批量写入

**当前代码**:
```rust
// 每个布尔结果单独 UPDATE
update_booled_result(&inst_relate_id, &mesh_id, &mesh).await?;
```

**优化后**:
```rust
struct BoolResult {
    inst_relate_id: String,
    mesh_id: String,
    aabb_hash: u64,
    success: bool,
}

// 收集所有结果
let results: Vec<BoolResult> = process_all_queries(queries).await;

// 批量 SQL
let sql = results.iter().map(|r| {
    if r.success {
        format!("UPDATE {} SET bool_status='Success', aabb=aabb:⟨{}⟩;", 
                r.inst_relate_id, r.aabb_hash)
    } else {
        format!("UPDATE {} SET bool_status='Failed';", r.inst_relate_id)
    }
}).collect::<Vec<_>>().join("\n");

SUL_DB.query(&sql).await?;
```

**预期收益**: 减少数据库往返 10-50x

### 方案 4：预加载流水线 ⭐

**目标**: I/O 与计算并行，形成流水线

**改造位置**: `manifold_bool.rs` - `apply_insts_boolean_manifold`

**实现**:
```rust
use flume;

pub async fn apply_insts_boolean_manifold_pipeline(
    queries: Vec<ManiGeoTransQuery>,
    replace_exist: bool,
) -> anyhow::Result<()> {
    let (tx, rx) = flume::bounded(16);
    
    // 预加载任务（I/O 密集）
    let preload_handle = tokio::spawn(async move {
        for query in queries {
            let pos_manifolds = preload_positive_manifolds(&query).await;
            let neg_manifolds = preload_negative_manifolds(&query).await;
            if tx.send_async((query, pos_manifolds, neg_manifolds)).await.is_err() {
                break;
            }
        }
    });
    
    // 布尔计算任务（CPU 密集）
    while let Ok((query, pos, neg)) = rx.recv_async().await {
        apply_boolean_with_preloaded(query, pos, neg, replace_exist).await?;
    }
    
    preload_handle.await?;
    Ok(())
}
```

**预期收益**: 1.3-1.8x（I/O 等待时间与计算重叠）

### 方案 5：Boolean Worker 并发增强 ⭐⭐

**目标**: 在 Worker 层面实现并发调度

**改造位置**: `mesh_generate.rs` - `run_boolean_worker`

**实现**:
```rust
use tokio::sync::Semaphore;

pub async fn run_boolean_worker_parallel(
    db_option: Arc<DbOption>,
    batch_size: usize,
    concurrency: usize,
) -> anyhow::Result<()> {
    let semaphore = Arc::new(Semaphore::new(concurrency));
    let relation_targets = query_relation_targets_combined().await?;
    
    loop {
        let cata_refnos = query_pending_cata_boolean(batch_size, false).await?;
        let inst_refnos = query_pending_inst_boolean(batch_size, false, &relation_targets).await?;
        
        if cata_refnos.is_empty() && inst_refnos.is_empty() {
            break;
        }
        
        // 分块并发执行
        let mut handles = Vec::new();
        
        for chunk in cata_refnos.chunks(20) {
            let permit = semaphore.clone().acquire_owned().await?;
            let db_opt = db_option.clone();
            let chunk = chunk.to_vec();
            handles.push(tokio::spawn(async move {
                let _permit = permit;
                apply_cata_neg_boolean_manifold(&chunk, false).await
            }));
        }
        
        for chunk in inst_refnos.chunks(20) {
            let permit = semaphore.clone().acquire_owned().await?;
            let db_opt = db_option.clone();
            let chunk = chunk.to_vec();
            handles.push(tokio::spawn(async move {
                let _permit = permit;
                apply_insts_boolean_manifold(&chunk, false).await
            }));
        }
        
        futures::future::try_join_all(handles).await?;
    }
    
    Ok(())
}
```

**预期收益**: 2-4x

## 3. 优化方案对比

| 方案 | 复杂度 | 预期收益 | 风险 | 优先级 |
|------|--------|----------|------|--------|
| 并发执行布尔 | 中 | 4-6x | 内存增加 | ⭐⭐⭐ |
| Mesh 缓存池 | 低 | 1.5-2x | 内存占用 | ⭐⭐⭐ |
| 批量 DB 写入 | 低 | 1.2-1.5x | 低 | ⭐⭐ |
| 预加载流水线 | 中 | 1.3-1.8x | 复杂度 | ⭐ |
| Worker 并发增强 | 中 | 2-4x | 需测试 | ⭐⭐ |

## 4. 实施路线

### Phase 1: 短期优化（1-2 天）

- [ ] **实现 Mesh 缓存池**（方案 2）
  - 改动最小，收益明显
  - 添加 `MANIFOLD_CACHE` 全局缓存
  - 在 `booleans_meshes_in_db` 结束后调用 `clear_manifold_cache()`

- [ ] **批量 DB 写入**（方案 3）
  - 修改 `apply_boolean_for_query` 返回结果而非直接写入
  - 在 `apply_insts_boolean_manifold` 收集结果后批量更新

### Phase 2: 中期优化（3-5 天）

- [ ] **并发执行布尔运算**（方案 1）
  - 添加 `futures` 依赖（如未添加）
  - 使用 `buffer_unordered` 实现并发
  - 添加并发数配置项到 `DbOption`

- [ ] **添加性能监控**
  - 布尔运算耗时日志
  - Mesh 加载统计（缓存命中率）
  - 并发任务数监控

### Phase 3: 长期优化

- [ ] **预加载流水线**（方案 4）
  - 重构为生产者-消费者模式
  - I/O 与计算解耦

- [ ] **GPU 加速探索**
  - Manifold 支持 CUDA
  - 评估 GPU 布尔运算收益
