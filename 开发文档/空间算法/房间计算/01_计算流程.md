# 房间计算流程

## 1. 整体计算流程

房间计算的目标是确定哪些构件位于由面板围成的房间空间内。计算分为两个阶段：

### 1.1 入口函数

```rust
// src/fast_model/room_model.rs
pub async fn build_room_relations(db_option: &DbOption) -> anyhow::Result<RoomBuildStats>
```

### 1.2 主要步骤

```
1. 构建房间-面板映射关系
   └── build_room_panels_relate(&room_key_words)

2. 对每个房间执行计算
   └── compute_room_relations(mesh_dir, room_panel_map, exclude_refnos, options)

3. 保存房间关系到数据库
   └── save_room_relate(panel_refno, within_refnos, room_num)
```

## 2. 粗算阶段（Coarse Calculation）

### 2.1 目的

快速筛选出可能与房间相关的构件，排除明显不在房间内的构件。

### 2.2 实现原理

使用 **SQLite RTree 空间索引** 查询与面板 AABB 重叠的所有构件：

```rust
// src/fast_model/room_model.rs: cal_room_refnos()

// 步骤 3：粗算 - 通过空间索引查询候选构件
let panel_aabb: Aabb = panel_geom_insts[0].world_aabb.into();

let candidates = sqlite::query_overlap(
    &panel_aabb,    // 面板的 AABB
    None,           // 无额外过滤
    Some(1000),     // 最大返回数量
    &exclude_list   // 排除列表（其他面板）
)?;
```

### 2.3 时间复杂度

- **查询复杂度**: O(log n + k)，其中 n 是索引中的总条目数，k 是结果数量
- **典型耗时**: 100-200ms

### 2.4 输出

候选构件列表，通常包含 100-500 个构件（取决于房间大小和周围构件密度）。

## 3. 精算阶段（Fine Calculation）

### 3.1 目的

对粗算筛选出的候选构件进行精确的几何包含测试。

### 3.2 关键点提取

从构件的 AABB 中提取 27 个关键点用于测试：

```rust
// src/fast_model/room_model.rs: extract_aabb_key_points()

fn extract_aabb_key_points(aabb: &Aabb) -> Vec<Point<Real>> {
    let mut points = Vec::with_capacity(27);
    let vertices = aabb.vertices();  // 8 个顶点

    // 1. 8 个顶点
    points.extend(vertices.iter().map(|v| *v));

    // 2. 中心点
    points.push(aabb.center());

    // 3. 6 个面中心
    // ...

    // 4. 12 条边中点
    // ...

    points  // 共 27 个关键点
}
```

### 3.3 射线投射法

**核心算法**：从测试点向多个方向发射射线，如果相反方向都有交点，则点在网格内部。

```rust
// src/fast_model/room_model.rs: is_point_inside_mesh_raycast()

fn is_point_inside_mesh_raycast(point: &Point<Real>, tri_mesh: &TriMesh) -> bool {
    let identity = Isometry::identity();

    // 向 +Z 和 -Z 方向发射射线
    let ray_pos_z = Ray::new(*point, Vector::new(0.0, 0.0, 1.0));
    let ray_neg_z = Ray::new(*point, Vector::new(0.0, 0.0, -1.0));

    let hit_pos_z = tri_mesh.cast_ray(&identity, &ray_pos_z, Real::MAX, true);
    let hit_neg_z = tri_mesh.cast_ray(&identity, &ray_neg_z, Real::MAX, true);

    // 如果 Z 方向两边都有交点，点在网格内部
    if hit_pos_z.is_some() && hit_neg_z.is_some() {
        return true;
    }

    // 备用检测：向 +X/-X 或 +Y/-Y 方向检测
    // ...
}
```

### 3.4 投票策略

使用阈值策略判断构件是否在房间内：

```rust
// src/fast_model/room_model.rs: is_geom_in_panel()

// 阈值策略：超过 50% 的关键点在面板内
let threshold = (total_points as f32 * 0.5) as usize;
points_inside >= threshold
```

### 3.5 容差处理

对于接近表面的点，使用距离容差：

```rust
// 回退到距离检测：如果点非常接近表面，也认为在内部
let projection = panel_tri_mesh.project_point(&Isometry::identity(), point, true);
let distance_sq = (projection.point - point).norm_squared();
if distance_sq <= tolerance_sq {
    points_inside += 1;
}
```

### 4.3 TriMesh 构建

```rust
mesh.get_tri_mesh_with_flag(
    (world_trans * inst.transform).to_matrix(),
    TriMeshFlags::ORIENTED | TriMeshFlags::MERGE_DUPLICATE_VERTICES,
)
```

**重要标志**:
- `ORIENTED`: 启用有向网格，计算 pseudo-normals
- `MERGE_DUPLICATE_VERTICES`: 合并重复顶点，优化性能

## 5. 结果保存

### 5.1 数据库关系

```sql
-- 创建房间-构件关系
RELATE pe:{panel} -> room_relate:{id} -> pe:{component}
SET room_num = '{room_number}',
    confidence = 0.9,
    created_at = time::now();
```

### 5.2 批量执行

为提高性能，SQL 语句会累积后批量执行：

```rust
let mut sql_statements = Vec::new();

for refno in within_refnos {
    let sql = format!(
        "relate {}->room_relate:{}->{}  set room_num='{}', confidence=0.9;",
        panel_refno.to_pe_key(),
        relation_id,
        refno.to_pe_key(),
        room_num
    );
    sql_statements.push(sql);
}

// 批量执行
let combined_sql = sql_statements.join("\n");
db.query(&combined_sql).await?;
```

## 6. 并发处理

### 6.1 并发配置

```rust
// 默认并发数，可通过环境变量配置
fn default_room_concurrency() -> usize {
    std::env::var("ROOM_RELATION_CONCURRENCY")
        .ok()
        .and_then(|v| v.parse::<usize>().ok())
        .filter(|c| *c > 0)
        .unwrap_or(4)
}
```

### 6.2 任务分发

```rust
// 并发处理多个房间
for chunk in room_panel_map.chunks(concurrency) {
    let handles: Vec<_> = chunk
        .iter()
        .map(|(room_num, panel_refno, panel_refnos)| {
            tokio::spawn(async move {
                process_single_room(...)
            })
        })
        .collect();

    futures::future::join_all(handles).await;
}
```
