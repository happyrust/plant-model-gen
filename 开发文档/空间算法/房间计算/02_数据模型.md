# 房间计算数据模型

## 1. PDMS 层级结构

### 1.1 房间相关层级

```
SITE (站点)
└── ZONE (区域)
    └── FRMW (框架/房间)
        └── SBFR (子框架)
            └── PANE (面板)
                └── shapes (几何形状)
```

### 1.2 房间识别

房间通过 `FRMW` 节点识别，使用配置的关键字进行匹配：

```rust
// DbOption.toml
room_key_words = ["FRMW", "ROOM", "舱室"]
```

## 2. 核心数据结构

### 2.1 GeomInstQuery

几何实例查询结果，包含构件的几何信息：

```rust
pub struct GeomInstQuery {
    pub refno: RefnoEnum,           // 构件引用号
    pub world_trans: PlantTransform, // 世界变换矩阵
    pub world_aabb: RStarBoundingBox, // 世界坐标系 AABB
    pub insts: Vec<ModelHashInst>,   // 几何实例列表
}
```

### 2.2 ModelHashInst

几何哈希实例，关联到具体的 mesh 文件：

```rust
pub struct ModelHashInst {
    pub geo_hash: String,           // 几何哈希 (用于查找 mesh 文件)
    pub transform: PlantTransform,   // 实例变换
    pub aabb: Option<Aabb>,          // 局部 AABB
}
```

### 2.3 RoomBuildStats

房间构建统计信息：

```rust
pub struct RoomBuildStats {
    pub total_rooms: usize,      // 处理的房间总数
    pub total_panels: usize,     // 处理的面板总数
    pub total_components: usize, // 找到的构件总数
    pub build_time_ms: u64,      // 构建耗时（毫秒）
    pub cache_hit_rate: f32,     // 缓存命中率
    pub memory_usage_mb: f32,    // 内存使用（MB）
}
```

## 3. 数据库表结构

### 3.1 pe 表（元件）

```sql
DEFINE TABLE pe SCHEMAFULL;

DEFINE FIELD refno ON pe TYPE string;        -- 引用号
DEFINE FIELD noun ON pe TYPE string;         -- 类型 (FRMW, PANE, TEE, etc.)
DEFINE FIELD name ON pe TYPE option<string>; -- 名称
DEFINE FIELD parent ON pe TYPE option<record<pe>>; -- 父节点
DEFINE FIELD children ON pe TYPE option<array<record<pe>>>; -- 子节点
```

### 3.2 inst_relate 表（实例关系）

```sql
DEFINE TABLE inst_relate SCHEMAFULL;

DEFINE FIELD refno ON inst_relate TYPE string;
DEFINE FIELD world_trans ON inst_relate TYPE object;  -- 世界变换
DEFINE FIELD world_aabb ON inst_relate TYPE object;   -- 世界 AABB
DEFINE FIELD geo_hash ON inst_relate TYPE option<string>; -- 几何哈希
```

### 3.3 room_relate 表（房间关系）

```sql
DEFINE TABLE room_relate SCHEMAFULL TYPE RELATION
    IN record<pe>   -- 面板 (房间的组成部分)
    OUT record<pe>; -- 构件 (在房间内的设备/管道)

DEFINE FIELD room_num ON room_relate TYPE string;      -- 房间编号
DEFINE FIELD confidence ON room_relate TYPE float;     -- 置信度 (0-1)
DEFINE FIELD created_at ON room_relate TYPE datetime;  -- 创建时间
```

## 4. 空间索引结构

### 4.1 SQLite RTree 索引

```sql
CREATE VIRTUAL TABLE spatial_index USING rtree(
    id,                    -- refno 的整数形式
    min_x, max_x,          -- X 范围
    min_y, max_y,          -- Y 范围
    min_z, max_z           -- Z 范围

```

## 6. Mesh 文件结构

### 6.1 文件路径

```
assets/meshes/
├── lod_L0/
│   └── {geo_hash}_L0.mesh
├── lod_L1/
│   └── {geo_hash}_L1.mesh
├── lod_L2/
│   └── {geo_hash}_L2.mesh
└── lod_L3/
    └── {geo_hash}_L3.mesh
```

### 6.2 PlantMesh 结构

```rust
pub struct PlantMesh {
    pub vertices: Vec<Vec3>,      // 顶点列表
    pub indices: Vec<u32>,        // 三角形索引
    pub normals: Vec<Vec3>,       // 法向量
    pub uvs: Vec<Vec2>,           // UV 坐标
    pub wire_vertices: Vec<Vec3>, // 线框顶点
    pub edges: Vec<(u32, u32)>,   // 边列表
    pub aabb: Option<Aabb>,       // 包围盒
}
```

## 7. 查询示例

### 7.1 查询房间的面板

```sql
-- 递归查询 FRMW 下所有 PANE
SELECT value array::flatten(@.{1..2+collect}.children)[?noun='PANE'].id
FROM pe:24381_35269
```

### 7.2 查询面板的几何数据

```sql
SELECT
    refno,
    world_trans,
    world_aabb,
    (SELECT geo_hash, transform FROM ->inst_geo) as insts
FROM inst_relate
WHERE refno = '24381_35271'
```

### 7.3 查询房间内的构件

```sql
SELECT out.refno, out.noun, room_num
FROM room_relate
WHERE in = pe:24381_35271
```
