# 房间计算注意事项与问题排查

## 1. parry3d `is_inside` 不可靠

### 1.1 问题描述

`parry3d::TriMesh::project_point().is_inside` 对于某些封闭网格返回错误结果。

**症状**:
- 明确在网格内部的点，`is_inside` 返回 `false`
- 网格的 AABB 中心点也返回 `is_inside: false`

**验证条件**:
- 网格是有效的封闭流形（0 边界边）
- 绕向一致（所有法向量朝外）
- pseudo-normals 已正确计算（`pseudo_normals()` 返回 `Some`）

### 1.2 根本原因

parry3d 的 `project_point` 内部使用 pseudo-normals 判断内外，但对于某些复杂的非凸多边形挤压体，算法可能失效。

### 1.3 解决方案

**使用射线投射法替代 `is_inside`**:

```rust
fn is_point_inside_mesh_raycast(point: &Point<Real>, tri_mesh: &TriMesh) -> bool {
    let identity = Isometry::identity();

    // 向 +Z 和 -Z 方向发射射线
    let ray_pos_z = Ray::new(*point, Vector::new(0.0, 0.0, 1.0));
    let ray_neg_z = Ray::new(*point, Vector::new(0.0, 0.0, -1.0));

    let hit_pos_z = tri_mesh.cast_ray(&identity, &ray_pos_z, Real::MAX, true);
    let hit_neg_z = tri_mesh.cast_ray(&identity, &ray_neg_z, Real::MAX, true);

    // 如果两个方向都有交点，点在网格内部
    if hit_pos_z.is_some() && hit_neg_z.is_some() {
        return true;
    }

    // 备用检测：尝试 X 和 Y 方向
    // ...
}
```

### 1.4 验证方法

运行测试示例验证射线投射法：

```bash
cargo run --example room_tee_containment --features "gen_model,sqlite-index"
```

预期输出应包含：
```
--- 射线投射法验证 ---
从三通中心 [-124.8, -14983.8, 230.9] 发射射线:
   +Z 方向: Some("命中, 距离=2259.12")
   -Z 方向: Some("命中, 距离=200.88")
   两个方向都有交点: ✅
```

## 2. LOD 文件路径问题

### 2.1 问题描述

几何文件可能不存在于预期的 LOD 级别目录中。

**症状**:
- 精算返回 0 个构件
- 日志显示 "无法加载几何文件"

### 2.2 原因

`get_meshes_path()` 返回的路径已包含 LOD 子目录，但 `build_mesh_path()` 会再次添加 LOD 前缀，导致路径错误。

### 2.3 解决方案

使用 LOD 回退机制，并正确处理路径：

```rust
// 获取基础目录（去掉 LOD 子目录）
let base_mesh_dir = mesh_dir
    .parent()
    .map(|p| p.to_path_buf())
    .unwrap_or_else(|| mesh_dir.clone());

// 尝试多个 LOD 级别
let lod_levels = ["L0", "L1", "L2", "L3"];

for lod_level in lod_levels.iter() {
    let file_path = base_mesh_dir
        .join(format!("lod_{}", lod_level))
        .join(format!("{}_{}.mesh", geo_hash, lod_level));

    if file_path.exists() {
        // 加载并返回
    }
}
```

### 2.4 验证方法

检查 mesh 文件是否存在：

```bash
# 检查特定 geo_hash 的文件
ls -la /path/to/meshes/lod_L*/15544281351836005492_L*.mesh
```

## 3. TriMeshFlags 配置

### 3.1 必需的标志

```rust
TriMeshFlags::ORIENTED | TriMeshFlags::MERGE_DUPLICATE_VERTICES
```

- **ORIENTED**: 启用有向网格，计算 pseudo-normals
- **MERGE_DUPLICATE_VERTICES**: 合并重复顶点，减少内存和提高性能

### 3.2 可选的标志

```rust
TriMeshFlags::FIX_INTERNAL_EDGES  // 修复内部边
TriMeshFlags::HALF_EDGE_TOPOLOGY  // 计算半边拓扑
```

**注意**: 添加更多标志会增加构建时间，但不一定改善 `is_inside` 的准确性。

### 3.3 验证 pseudo-normals

```rust
if let Some(pn) = tri_mesh.pseudo_normals() {
    println!("pseudo_normals 可用");
    println!("顶点数: {}", pn.vertices_pseudo_normal.len());
} else {
    println!("pseudo_normals 不可用 - 需要检查 TriMeshFlags");
}
```

## 4. 性能优化建议

### 4.1 缓存配置

```rust
// 设置合理的缓存大小
if cache.len() > 2000 {
    cleanup_geometry_cache(&cache).await;
}
```

**建议**: 根据可用内存调整缓存大小，通常 1000-3000 是合理范围。

### 4.2 并发配置

```bash
# 通过环境变量设置并发数
export ROOM_RELATION_CONCURRENCY=8
```

**建议**: 设置为 CPU 核心数的 50-100%。

### 4.3 批量查询

避免逐个查询构件，尽量使用批量查询：

```rust
// 好的做法
let all_insts = aios_core::query_insts(&refno_list, true).await?;

// 避免
for refno in refno_list {
    let inst = aios_core::query_insts(&[refno], true).await?;
}
```

## 5. 常见问题排查

### 5.1 精算结果为空

**检查步骤**:
1. 确认粗算有候选结果
2. 检查 mesh 文件是否存在
3. 验证 TriMesh 构建是否成功
4. 检查关键点是否在 AABB 范围内

**调试代码**:
```rust
println!("候选数量: {}", candidates.len());
println!("面板 AABB: {:?}", panel_aabb);
println!("构件 AABB: {:?}", candidate_aabb);
```

### 5.2 粗算结果过多

**原因**: 面板 AABB 过大，或房间跨越多层

**解决方案**:
- 增加排除列表
- 调整查询限制
- 细分大房间

### 5.3 计算时间过长

**检查步骤**:
1. 检查候选数量（> 500 可能过多）
2. 检查缓存命中率
3. 分析 mesh 文件大小

**优化建议**:
- 使用更低精度的 LOD（L0 < L1 < L2）
- 预热缓存
- 增加并发数

### 5.4 数据库连接问题

**症状**: 查询超时或返回空结果

**检查**:
```bash
# 验证 SurrealDB 连接
surreal sql --conn ws://localhost:8020 --ns 1516 --db AvevaMarineSample
```

### 6.3 mesh 拓扑分析

在代码中添加拓扑检查：

```rust
// 边界边检测
let mut edge_count: HashMap<(u32, u32), u32> = HashMap::new();
for tri in mesh.indices.chunks(3) {
    for i in 0..3 {
        let v1 = tri[i];
        let v2 = tri[(i + 1) % 3];
        let edge = if v1 < v2 { (v1, v2) } else { (v2, v1) };
        *edge_count.entry(edge).or_insert(0) += 1;
    }
}

let boundary_edges = edge_count.values().filter(|&&c| c == 1).count();
println!("边界边数: {} (0 = 封闭)", boundary_edges);
```

## 7. 版本兼容性

### 7.1 parry3d 版本

当前使用: `parry3d = "0.13"` (或更高)

**注意**: 不同版本的 `is_inside` 行为可能不同，升级时需重新验证。

### 7.2 SurrealDB 版本

## 8. 已知限制

### 8.1 非凸多边形

射线投射法对于极端非凸的多边形（如星形）可能产生误判。

**缓解措施**: 使用多方向射线检测（Z、X、Y）。

### 8.2 薄壁结构

对于非常薄的面板（厚度 < 容差），可能出现边界情况。

**缓解措施**: 适当调整容差参数。

### 8.3 大型房间

跨越多层的大型房间可能导致候选数量过多。

**缓解措施**: 分层处理或细分房间。
